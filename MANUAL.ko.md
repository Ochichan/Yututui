# YuTuTui! 사용 설명서

[English](MANUAL.md) · **한국어** · [日本語](MANUAL.ja.md)

터미널이 낯선 분들을 위해, 천천히 하나하나 짚어가는 안내서입니다. 전문 용어 없이, 모든 단계를 풀어 씁니다. (터미널이 익숙하다면 [README](README.ko.md)에 빠른 버전이 있어요.)

무엇보다 먼저 이것 하나만: **앱 안에서 언제든 `?` 를 누를 수 있습니다.** 모든 키가 정리된 치트시트가 뜨고, 항상 *내 설정* 그대로 반영돼요. 이 설명서에서 하나만 기억한다면 `?` 를 기억하세요.

---

## 1. 첫걸음

### 설치하고 열기

[README](README.ko.md#설치)의 한 줄 설치를 따라 하세요. Windows에서는 이제 시작 메뉴의
**YuTuTui!** 를 누르면 Windows Terminal과 플레이어가 함께 열립니다. Tray 아이콘의
**플레이어 열기(Open Player)** 도 같은 동작입니다. 다른 OS에서는 터미널 앱을 엽니다 —
명령어를 입력하는 그 창이에요:

- **macOS** — *터미널(Terminal)* 앱 (iTerm2가 있다면 그것도 좋아요)
- **Windows** — 시작 메뉴의 *Windows Terminal*
- **Linux** — 늘 쓰시는 그거요

이렇게 입력하고 Enter:

```sh
ytt
```

실행은 이게 전부입니다. 창 안에 플레이어가 나타나요.

첫 실행에는 10초 동안 검색 위치를 알려줍니다. 표시된 키(기본 `s`)를 누르거나 **검색**을
클릭하세요. mpv, yt-dlp, ffmpeg가 빠졌다면 설치 카드의 복사/안내 버튼을 이용하고 설치 뒤
**다시 확인**을 누르세요. 기술적인 상세 진단은 계속 `ytt doctor`에서 볼 수 있습니다.

### 첫 곡 재생하기

1. **`s`** 를 누르면 검색창이 열립니다.
2. 곡이나 가수 이름을 입력하고 **`Enter`**. 모든 텍스트 입력칸에서 **`←`** / **`→`** 로 한 글자씩, **`Ctrl+←`** / **`Ctrl+→`** 로 단어씩 커서를 옮길 수 있고, **`Backspace`** 로 한 글자, **`Ctrl+Backspace`** 로 이전 단어를 지울 수 있습니다.
3. **`↓`** 로 내려가서 원하는 곡에 **`Enter`**.

음악이 나옵니다. 만약 오류가 났다면 터미널에 `ytt doctor` 를 입력해 보세요 — 설정을 점검하고 뭘 고쳐야 하는지 쉬운 말로 알려줍니다.

**완전 처음이신가요?** **설정 → 일반**에서 **비기너 모드**를 켜면, 다음 실행 때 각 부분을 하나하나 짚어주는 인터랙티브 단계별 안내가 함께 나옵니다.

### 다섯 개의 화면

YuTuTui!는 다섯 화면으로 이루어져 있고, 각각 키 하나 거리예요:

| 키 | 화면 | 용도 |
| --- | --- | --- |
| — | **플레이어** | 지금 재생 중 화면: 앨범 아트, 가사, 진행 바 |
| `s` | **검색** | 곡, 앨범, 가수, 방송국 찾기 |
| `l` | **라이브러리** | 즐겨찾기, 기록, 다운로드, 플레이리스트 |
| `o` | **설정** | 조절 가능한 모든 것 — 계정 연결도 여기 |
| `g` | **DJ Gem** | 말로 음악 부탁하기 *(선택 기능, 아래 참고)* |

`Esc` 는 대체로 지금 있는 곳에서 빠져나오는 키입니다. 마우스도 어디서나 통해요 — 아무거나 클릭하고, 휠을 돌리면 볼륨이 움직입니다.

### 플레이어 바는 어디든 따라와요

지금 재생 중인 곡의 컨트롤 — 제목, 진행 바, 재생 버튼, 상태줄 — 이 **모든** 화면 하단에
도킹된 바로 붙어 다닙니다. 검색이나 라이브러리에서 화면을 떠나지 않고 일시정지·탐색이
가능해요. `Shift+B` 키(또는 하단 마우스 안내 옆의 `▼` / `▲` 클릭)로 다른 화면에서는 바를 접어
공간을 되찾을 수 있고, 플레이어 화면에는 항상 남아 있습니다. 컨트롤이 플레이어 화면
상단에만 있던 예전 모습이 좋다면 *설정 › 일반 › 플레이어 바 위치* 를 **상단**으로 바꾸면
돼요.

창을 아주 작게(대략 32×14 칸 미만) 줄이면 앱 전체가 작은 미니플레이어 — 제목, 진행 바,
재생 버튼 — 로 바뀌고, 창이 다시 커지면 곧바로 원래 화면으로 돌아옵니다. 따로 설정할
것 없이 창 크기를 그대로 따라가요.

## 2. 평소의 음악 생활 (일반 모드)

### 실제로 자주 쓰는 키

| 키 | 동작 |
| --- | --- |
| `Space` | 재생 / 일시정지 |
| `,` / `.` | 이전 / 다음 곡 |
| `←` / `→` | 되감기 / 빨리 감기 |
| `↑` / `↓` | 볼륨 |
| `f` | 지금 곡을 좋아요 / 싫어요 / 평가 없음 순서로 전환 |
| `x` / `r` | 셔플 / 반복 모드 전환 |
| `c` | 큐 보기 (다음에 나올 곡들) |
| `Shift+L` | 음악에 맞춰 흐르는 싱크 가사; 보이는 행 클릭으로 해당 시점 탐색 |
| `z` / `Shift+Z` | 가사를 0.1초 앞당김 / 늦춤 |
| `v` | 떠 있는 창으로 뮤직비디오 |
| `Ctrl+Q` | 종료 |

싱크 가사가 로드되면 오른쪽 아래에 **`[ − 0.0s + ]`** 가 3초 동안 나타납니다. **`[±]`** 로 접힌 뒤에는 핸들을 클릭해 다시 3초간 펼칠 수 있고, **`−/+`** 로 가사를 0.1초씩 앞당기거나 늦춰 미세 조정합니다. 보이는 가사 행을 클릭하면 해당 싱크 시점으로 탐색합니다.

**소리가 엉뚱한 장치로 나오나요?** **설정 → 재생 → 오디오 출력**에서 앱이 감지한 출력 장치 중 하나를 고르세요; **오디오 백엔드**에서는 그 아래 mpv 오디오 옵션을 열 수 있습니다.

마우스는 어디서나 통합니다: 행을 **우클릭**하면 컨텍스트 메뉴가 열려요(제스처는 `config.json` 의 `mouse_bindings` 에서 바꿀 수 있습니다).

### 라이브러리

**`l`** 을 누르세요. 라이브러리에는 다섯 탭이 있습니다: **전체**, **즐겨찾기**, **기록**, **다운로드**, **플레이리스트**. 즐겨찾고, 듣고, 내려받고, 모은 모든 것이 이 중 한 곳에 들어가요. **`n`** 을 누르면 나만의 새 플레이리스트를 만들 수 있습니다.

### 다운로드 — 오프라인으로 간직하기

아무 곡에서나 **`d`** 를 누르면 제대로 된 음악 파일(커버 아트와 제목 포함)로 음악 폴더에 저장되고, 라이브러리 → 다운로드에 나타납니다. **`Shift+D`** 는 목록이나 플레이리스트를 통째로 내려받아요. 다운로드한 곡은 인터넷 없이 재생되고 — 로컬 덱(4장)의 재료가 됩니다.

### DJ Gem *(선택)*

DJ Gem은 앱에 내장된 음악 두뇌입니다. 선택 기능이고 Google의 무료 Gemini API 키가 필요해요 — 나머지 기능은 키 없이도 전부 동작합니다.

- **끝없는 스테이션:** **`Ctrl+R`** 을 누르면 지금 듣는 분위기에 맞는 곡으로 큐를 계속 채워줍니다. **`w`** 를 누르면 각 곡을 왜 골랐는지 쉬운 말로 설명해 줘요.
- **말로 부탁하기:** **`g`** 를 누르고 *"조용한 피아노곡 틀어줘"*, *"비 오는 날 플레이리스트 만들어줘"* 처럼 입력하면 — 플레이리스트를 라이브러리에 바로 만들어 주기도 합니다.

켜는 법: Google에서 무료 Gemini API 키를 받아 **설정 → DJ Gem** 에 붙여넣고 활성화하세요.

## 3. 라디오 모드 — 앱이 라디오 튜너가 됩니다

곡을 고르는 것조차 귀찮은 날이 있죠. 라디오 모드는 앱 전체를 인터넷 라디오 튜너로 바꿉니다 — 진짜 생방송 방송국이 수천 개예요.

### 들어가기 / 나오기

플레이어 화면에서 **`Alt+Shift+R`** 을 누르세요. *"라디오 모드로 전환할까요?"* 라고 물어보면 확인 — 그 순간 모든 게 바뀝니다. 색이 라디오 전용 테마로 바뀌는데(일부러 그래요 — 지금 어디에 있는지 한눈에 알 수 있게), 듣던 음악 큐는 그대로 고이 보관됐다가 돌아오면 복원됩니다.

**`Alt+Shift+R`** 을 다시 누르면 일반 음악 모드로 돌아옵니다. (한 가지 규칙: 라디오와 로컬 덱은 동시에 열 수 없어요 — 한쪽을 나온 뒤 다른 쪽에 들어가세요.)

### 방송국 찾기

**`s`** 를 눌러 곡을 찾듯 검색하세요 — 다만 지금은 **Radio Browser**, 인터넷 방송국의 거대한 공개 목록을 검색하는 겁니다. 장르("jazz"), 나라, 방송국 이름으로 찾아서 `Enter` 로 주파수를 맞추세요.

라디오 모드에서는 라이브러리(`l`)도 달라집니다: **라디오 즐겨찾기** 와 **라디오 기록** 두 탭만 보여요 — 좋아하는 방송국과 최근 들은 방송국이고, 음악 즐겨찾기와는 완전히 따로 관리됩니다. 방송국 위에서 **`f`** 를 누르면 즐겨찾기.

### 듣는 동안

생방송은 *생방송*이라 되감기가 없습니다. 연결이 잠깐 출렁여서 방송보다 뒤처지면 앱이 알려줘요 — *"Live: 25s behind"* — 그때 **`r`** 을 누르면 라이브 최전선으로 착 붙습니다.

백미는 이겁니다: 좋은 곡이 걸리면 **`i`** 를 누르세요. *지금 나오는 곡이 뭔지* 알려주는 작은 카드가 뜹니다 — 방송국이 내보내는 정보를 그대로 읽어서요. 카드 안에서:

- **`f`** — 식별된 곡을 *음악* 즐겨찾기에 저장 (일반 모드로 돌아가면 거기 있어요),
- **`g`** — DJ Gem에게 더 알려달라거나 비슷한 곡을 부탁.

**`Alt+Shift+E`** 에는 라디오 녹음 브라우저도 있습니다.

## 4. 로컬 덱 — 내 컴퓨터의 음악을, 근사하게

로컬 덱은 *내 컴퓨터에 있는* 음악을 위한 전용 플레이어입니다: 다운로드한 곡들과 원래 갖고 있던 오디오 파일들. 인터넷이 아예 없어도 됩니다.

### 들어가기 / 나오기

라이브러리(**`l`**)를 연 다음 **`Alt+Shift+L`** 을 누르세요. *"로컬 플레이어 모드로 전환할까요?"* 라고 물으면 확인 — 로컬 음악 탐색만을 위해 만들어진 몰입형 화면으로 들어갑니다. **`Alt+Shift+L`** 을 다시 누르면 나와요.

### 무엇이 보이나

로컬 덱은 다운로드 폴더를 스캔해서(*가수 / 앨범 / 곡* 구조를 이해합니다) 모든 것을 섹션으로 정리합니다. **숫자 키**로 섹션 사이를 오가세요:

**홈 · 곡 · 앨범 · 아티스트 · 장르 · 폴더 · 스마트 리스트 · 스캔 오류 · 가져오기 세션 · 인박스**

- **곡 / 앨범 / 아티스트 / 장르** — 내 컬렉션을 원하는 단면으로.
- **폴더** — 디스크에 놓인 모습 그대로 탐색.
- **스마트 리스트** — 자동으로 만들어지는 모음.
- **스캔 오류** — 스캐너가 읽지 못한 파일들. 조용히 사라지는 건 없습니다.
- **가져오기 세션 / 인박스** — Spotify 가져오기가 검토를 기다리는 곳 (다음 장!).

### 음악 넣기

- `d` / `Shift+D` 로 내려받은 모든 곡이 자동으로 여기 나타납니다.
- 설정에서 스캔할 폴더를 더 추가할 수 있어요 (로컬 덱 루트).
- Spotify 가져오기가 바로 이곳으로 다운로드해 줄 수도 있습니다 — 계속 읽어보세요.

## 5. Spotify에서 이사 오기 — 찬찬히, 전부 다

Spotify의 플레이리스트와 좋아요 표시한 곡들을 YuTuTui!로 가져올 수 있습니다. 조용히 때려 맞추는 일은 없어요: 모든 곡은 실제 제목·가수·앨범으로 매칭되고, 확실하지 않은 곡은 *당신이* 결정하도록 따로 모아둡니다.

**음악이 어디로 가나요?** 두 가지 선택지가 있습니다:

1. **앱 자체의 라이브러리 플레이리스트로** — 바로 되고, YouTube 계정이 필요 없습니다. 앱 안 가져오기가 이 방식이에요.
2. **진짜 YouTube Music 계정으로** (플레이리스트나 좋아요) — 명령어 방식이고, YouTube 로그인 쿠키가 필요합니다 (README 참고 자료 참조).

### 5a. 최초 1회 설정 (~5분, 무료)

이 설정이 왜 있는지 솔직하게 말씀드릴게요: Spotify는 자기네에 등록된 앱에게만 라이브러리 열람을 허락하는데, 개인용 앱 등록("Development Mode")은 앱 주인이 직접 명단에 올린 사람만 받아줍니다. 그래서 모두가 앱 하나를 같이 쓰는 대신, *각자 자기만의 작은 무료 앱을 하나 만드는* 겁니다 — 5분 걸리고, 돈 안 들고, 나만 쓰게 됩니다. 비밀번호 같은 건 없어요; ID 하나만 복사하면 됩니다.

1. 브라우저에서 **[developer.spotify.com/dashboard](https://developer.spotify.com/dashboard)** 에 들어가 평소 Spotify 계정으로 로그인합니다.
2. **Create app** 을 클릭.
3. **App name** 과 **App description** 은 아무거나 — `yututui` 면 충분해요.
4. **Redirect URIs** 에 정확히 이렇게 입력하고:

   ```
   http://127.0.0.1:9271/callback
   ```

   **Add** 를 클릭합니다. 글자 하나까지 정확해야 해요 — `localhost` 라는 단어가 아니라 숫자 `127.0.0.1` (Spotify가 localhost는 거부합니다), 끝에 슬래시도 붙이지 마세요. 이 주소는 "이 컴퓨터의 앱으로 돌아와라"는 뜻일 뿐, 아무것도 밖으로 나가지 않습니다.
5. **Which API/SDKs are you planning to use?** 에서 **Web API** 를 체크.
6. 약관에 동의하고 **Save**.
7. 새로 만든 앱 → **Settings** 에서 **Client ID**(길게 늘어선 영문+숫자)를 복사합니다. "Client secret"은 무시하세요 — 필요 없습니다.
8. 같은 설정 화면에서 **User Management** 를 열고 *자기 자신*을 추가합니다: 이름과 Spotify 계정 이메일. 이게 허용 명단이에요 — 이 단계를 빼먹으면 나중에 Spotify가 "403"이라고 답합니다.

끝. 다시는 안 해도 됩니다.

### 5b. YuTuTui!와 Spotify 연결하기

앱에서: **설정(`o`) → 계정 → Spotify** → Client ID 붙여넣기 → **Connect**. 브라우저에 Spotify 승인 페이지가 열리면 승인 — 앱에 연결됐다고 표시됩니다.

(타이핑이 편하다면 `ytt auth spotify --client-id <내-ID>` 도 같은 일을 합니다. 브라우저가 없는 컴퓨터에서는 승인 링크가 클립보드에 복사되고 `spotify_auth_url.txt` 에도 저장되니, 아무 기기에서나 열면 돼요.)

### 5c. 앱 안에서 가져오기

1. **설정 → 계정 → "Spotify에서 가져오기…"** 로 갑니다.
2. 내 Spotify 플레이리스트 목록이 뜹니다. 하나 고르세요.
3. **가져오기 모드**를 고릅니다 (바로 옆에 드롭다운이 있어요):

   | 모드 | 뜻 |
   | --- | --- |
   | **빠른 가져오기 (Fast playlist)** | 확실한 매치 *그리고* 안전한 근사 매치까지. 대부분의 곡이 바로 들어옵니다. |
   | **엄격한 가져오기 (Strict playlist)** | 앱이 확신하는 매치만; 나머지는 전부 내 검토 대기줄로. |
   | **검토 먼저 (Review first)** | 매칭만 하고 아직 아무것도 쓰지 않기 — 나중에 내가 전부 승인. |

4. 이게 답니다 — 가져오기는 **음악이 계속 나오는 동안 백그라운드에서** 돌아갑니다. 상태줄에 진행 상황이 보여요.

끝나면 이런 메시지가 뜹니다: *"가져오기 완료 … 라이브러리 → 플레이리스트에 저장됨"*. 플레이리스트는 거기 있고, 바로 재생됩니다.

### 5d. 남은 곡 검토하기

앱이 확신하지 못한 곡은 절대 멋대로 채워지지 않습니다 — **로컬 덱 → 가져오기 세션** (그리고 **인박스**)에서 기다려요. 로컬 덱(4장)으로 가서 세션을 열고 행을 하나씩 보세요: Spotify에 있던 곡과 가장 유력한 후보들이 나란히 보입니다. 맞아 보이는 것만 승인하거나 — **`Shift+A`** 를 눌러 매치된 후보를 한 번에 전부 승인할 수도 있어요. 각 행에서 다운로드를 다시 시도하거나 후보 링크를 열어 직접 귀로 확인할 수도 있습니다.

### 5e. 명령어 버전 *(선택)*

명령어가 편하다면 같은 기능을 더 많은 옵션과 함께 쓸 수 있습니다:

```sh
ytt transfer import <spotify-url-또는-id>    # 플레이리스트 → 내 YTM 계정 (쿠키 필요)
ytt transfer import liked --to likes         # Spotify 좋아요 → YTM 좋아요, 순서 유지
ytt transfer import <url> --to local:이름     # → 앱 자체 라이브러리 플레이리스트 (YTM 계정 불필요)
ytt transfer resume <job-id>                 # 중단된 지점부터 이어하기
ytt transfer backup --dir ~/music-backup     # 모든 플레이리스트를 파일로 백업
ytt transfer session <id>                    # 가져오기 세션 들여다보기
```

가져오기는 체크포인트가 있습니다: 레이트 리밋이든, 노트북 덮개를 닫았든, 정전이든 — 나중에 `resume` 하면 끊긴 곳부터 이어집니다.

### 뭔가 잘못되면

가장 흔한 문제들(403 "허용 목록 없음", INVALID_CLIENT, 포트 사용 중)은 전부 **[README 문제 해결](README.ko.md#문제-해결)** 에 한 줄 처방이 있습니다.

## 6. 내 개인 데이터 백업하기

YuTuTui!는 이식 가능한 설정과 음악 취향을 버전 정보가 들어간, 사람이 읽을 수 있는 JSON 파일 하나로 모아 줍니다. 앱에서는 **설정(`o`) → 일반 → 개인 데이터 내보내기**를 선택하세요. 컴퓨터의 일반 **다운로드** 폴더에 저장한 뒤 완성된 파일명을 알려줍니다.

터미널에서도 똑같이 할 수 있습니다:

```sh
ytt data export                         # OS의 다운로드 폴더에 저장
ytt data export --to ~/existing-folder # 이미 있는 다른 폴더를 선택
```

`--to` 뒤에는 파일명이 아니라 **이미 존재하는 폴더**를 지정해야 합니다. YuTuTui!가 그 폴더를 새로 만들지 않으며, OS의 다운로드 폴더를 찾지 못했을 때 현재 폴더에 몰래 저장하지도 않습니다. 그럴 때는 `--to`로 폴더를 직접 지정하세요.

일반 기본 `ytt` 앱이나 데몬이 실행 중이면 CLI는 오래됐을 수 있는 디스크 파일 대신 그 프로세스의 최신 메모리 상태를 받아옵니다. 구버전·손상된 descriptor·살아 있지만 연결되지 않는 소유자·판단이 모호한 상태에서는 디스크로 몰래 전환하지 않고 내보내기를 중단합니다. 광고된 endpoint가 확실히 오래됐고 배타 데이터 잠금으로 어떤 프로세스도 저장소를 소유하지 않음이 확인될 때만 오프라인 스냅샷으로 안전하게 복구합니다. `--new-instance` 플레이어가 함께 열려 있어도 CLI는 광고된 기본 소유자만 내보냅니다. 각 보조 세션은 해당 설정 화면에서 따로 내보내세요. 오프라인 CLI는 보조 세션을 포함해 현재 버전의 ytt 소유자를 모두 닫기 전에는 디스크 저장소를 읽지 않습니다.

**포함되는 것:** 비밀값을 제거한 이식 가능한 설정, 즐겨찾기, 감상·라디오 기록, 라이브러리 플레이리스트, 안전한 곡 메타데이터와 공개 카탈로그 ID, 그리고 내 취향을 나타내는 추천 신호·아티스트 선호도·스테이션 선호 설정.

**의도적으로 제외되는 것:**

- 인증 쿠키, API 키, OAuth 토큰과 계정 식별값
- 파일시스템 경로와 기기별 오디오 설정
- 재생·원본·아트워크·라디오 스트림 URL
- 다운로드하거나 녹음한 음악, 다운로드 manifest와 미디어 sidecar
- 전송 대기 중인 스크로블, 전송 작업·리포트와 세션 큐
- AI 사용 데이터, 생성 캐시, 아트워크 캐시와 앱 로그
- 관리형 도구의 바이너리·경로, 데스크톱 창 배치와 복구 백업

내보낸 파일은 **암호화되지 않습니다.** 비밀번호나 토큰은 없지만 개인적인 감상 기록은 들어 있으므로, 업로드하거나 공유하기 전에 개인 파일로 취급하세요. 현재 버전은 데이터 **내보내기만** 지원하며 가져오기나 복원 명령은 아직 없습니다.

YuTuTui!는 기존 파일을 덮어쓰지 않고 현재 사용자만 읽을 수 있는 새 파일을 만듭니다. 신뢰하지 않는 로컬 계정이 완성 경로를 만들거나 교체·삭제할 수 있는 저장 폴더도 거부합니다. 대상 파일시스템이 이런 비공개 권한이나 ACL을 적용하고 확인할 수 없다면, 여러 계정이 읽을 수 있는 복사본을 남기는 대신 내보내기를 실패 처리합니다.

## 7. 뭔가 잘못됐을 때 (어디서든)

언제나 첫 단계: 앱을 끄고 이걸 실행하세요.

```sh
ytt doctor
```

보조 프로그램을 전부 점검하고, 뭐가 빠졌고 어떻게 구하는지 정확히 알려줍니다. 그 밖의 모든 것 — 재생 안 되는 곡, 안 보이는 앨범 아트, 스크로블, Spotify 오류 — 은 **[README 문제 해결 표](README.ko.md#문제-해결)** 에 증상별로 정리되어 있어요.

그래도 막히면? [이슈를 열고](https://github.com/Ochichan/Yututui/issues) 본 것을 그대로 적어주세요 — 운영체제도 함께요.
