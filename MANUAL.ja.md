# YuTuTui! マニュアル

[English](MANUAL.md) · [한국어](MANUAL.ko.md) · **日本語**

ターミナルに慣れていない方のための、ゆっくり丁寧なガイドです。専門用語なしで、すべての手順を一つずつ書きます。（ターミナルが得意なら [README](README.ja.md) に速い版があります。）

何よりも先にこれだけ: **アプリの中ではいつでも `?` を押せます。** すべてのキーをまとめたチートシートが開き、常に*あなたの設定*どおりに表示されます。このマニュアルから一つだけ覚えるなら、`?` を覚えてください。

---

## 1. はじめの一歩

### インストールして開く

[README](README.ja.md#インストール) の一行インストールに従ってください。Windows では
スタートメニューの **YuTuTui!** を選ぶと、Windows Terminal とプレイヤーが一緒に
開きます。Tray アイコンの **Open Player** も同じ動作です。他の OS ではターミナル
アプリを開きます — コマンドを打ち込むあのウィンドウです:

- **macOS** — *ターミナル (Terminal)* アプリ（iTerm2 があればそれでも）
- **Windows** — スタートメニューの *Windows Terminal*
- **Linux** — いつものあれです

こう入力して Enter:

```sh
ytt
```

起動はこれだけ。ウィンドウの中にプレイヤーが現れます。

初回起動では 10 秒間 Search の場所を案内します。表示されたキー（通常は `s`）を押すか
**Search** をクリックしてください。mpv、yt-dlp、ffmpeg が不足している場合は、案内
カードのコピー/ガイドボタンを使い、インストール後に **再確認** を選びます。技術的な
詳細は引き続き `ytt doctor` で確認できます。

### 最初の一曲を再生する

1. **`s`** を押すと検索ボックスが開きます。
2. 曲名かアーティスト名を入力して **`Enter`**。入力中は **`Backspace`** で一文字、**`Ctrl+Backspace`** で前の単語を削除できます。
3. **`↓`** で結果を下り、聴きたい曲で **`Enter`**。

音楽が流れます。もしエラーが出たら、ターミナルで `ytt doctor` と打ってみてください — 設定を点検して、直すべきところをやさしい言葉で教えてくれます。

**まったくの初めてですか？** **設定 → 一般** で **ビギナーモード** をオンにすると、次の起動から各部分を一つずつ案内してくれるインタラクティブな手ほどきが付きます。

### 五つの画面

YuTuTui! は五つの画面でできていて、それぞれキー一つの距離にあります:

| キー | 画面 | 用途 |
| --- | --- | --- |
| — | **プレイヤー** | 再生中の画面: アルバムアート、歌詞、シークバー |
| `s` | **検索** | 曲・アルバム・アーティスト・ラジオ局を探す |
| `l` | **ライブラリ** | お気に入り、履歴、ダウンロード、プレイリスト |
| `o` | **設定** | 調整できるすべて — アカウント接続もここ |
| `g` | **DJ Gem** | 言葉で音楽を頼む *（任意機能、下記参照）* |

`Esc` はだいたい「今いる場所から抜ける」キーです。マウスもどこでも使えます — 何でもクリックでき、ホイールを回すと音量が動きます。

### プレイヤーバーはどこへでもついてきます

再生中の曲のコントロール — タイトル、シークバー、再生ボタン、ステータス行 — は
**すべての**画面の下部にドッキングされたバーとして表示されます。検索やライブラリの画面を
離れずに一時停止やシークができます。`Shift+B` キー(またはフッターのマウス案内の隣の `▼` / `▲`
をクリック)で、プレイヤー以外の画面ではバーを畳んで行を取り戻せます。プレイヤー画面には
常に残ります。コントロールがプレイヤー画面の上部だけにあった以前の見た目が良ければ、
*設定 › 一般 › プレイヤーバーの位置* を **上部** に切り替えてください。

ウィンドウをうんと小さく(おおよそ 32×14 セル未満)すると、アプリ全体が小さな
ミニプレイヤー — タイトル、シークバー、再生ボタン — に変わり、ウィンドウが大きくなれば
すぐ元のレイアウトに戻ります。設定は不要で、ウィンドウサイズに自動で追従します。

## 2. ふだんの音楽（通常モード）

### 実際によく使うキー

| キー | 動作 |
| --- | --- |
| `Space` | 再生 / 一時停止 |
| `,` / `.` | 前 / 次の曲 |
| `←` / `→` | 巻き戻し / 早送り |
| `↑` / `↓` | 音量 |
| `f` | いまの曲を高評価 / 低評価 / 未評価の順に切り替え |
| `x` / `r` | シャッフル / リピート切り替え |
| `c` | キューを見る（次に流れる曲たち） |
| `Shift+L` | 音楽に合わせて流れる同期歌詞。表示中の行をクリックしてその時点へシーク |
| `z` / `Shift+Z` | 歌詞を 0.1 秒早く / 遅く |
| `v` | 浮かぶウィンドウでミュージックビデオ |
| `Ctrl+Q` | 終了 |

同期歌詞が読み込まれると、右下に **`[ − 0.0s + ]`** が 3 秒間表示されます。**`[±]`** に折りたたまれた後はハンドルをクリックして再び 3 秒間開き、**`−/+`** で歌詞を 0.1 秒ずつ早く / 遅く微調整できます。表示中の歌詞行をクリックすると、その同期位置へシークします。

**音が違うスピーカーから出る？** **設定 → 再生 → オーディオ出力** で、アプリが検出した出力デバイスから選べます; **オーディオバックエンド** ではその下の mpv オーディオオプションを開けます。

マウスはどこでも使えます: 行を**右クリック**するとコンテキストメニューが開きます（ジェスチャーは `config.json` の `mouse_bindings` で変更できます）。

### ライブラリ

**`l`** を押してください。ライブラリには五つのタブがあります: **すべて**、**お気に入り**、**履歴**、**ダウンロード**、**プレイリスト**。お気に入りにした曲、聴いた曲、保存した曲、集めた曲 — 全部このどれかに入ります。**`n`** で自分の新しいプレイリストを作れます。

### ダウンロード — オフラインで手元に

どの曲の上でも **`d`** を押せば、ちゃんとした音楽ファイル（カバーアートとタイトル入り）としてミュージックフォルダに保存され、ライブラリ → ダウンロードに現れます。**`Shift+D`** はリストやプレイリストを丸ごと保存。ダウンロードした曲はインターネットなしで再生でき — ローカルデッキ（第4章）の素材にもなります。

### DJ Gem *（任意）*

DJ Gem はアプリ内蔵の音楽ブレインです。任意機能で、Google の無料 Gemini API キーが必要です — それ以外の機能はキーなしで全部動きます。

- **果てしないステーション:** **`Ctrl+R`** を押すと、いま聴いている雰囲気に合う曲でキューを満たし続けます。**`w`** を押すと、それぞれの曲を選んだ理由をやさしい言葉で説明してくれます。
- **言葉で頼む:** **`g`** を押して *「静かなピアノ曲をかけて」*、*「雨の日プレイリストを作って」* のように入力 — プレイリストをライブラリに直接作ってくれたりもします。

オンにするには: Google で無料の Gemini API キーを取得し、**設定 → DJ Gem** に貼り付けて有効化してください。

## 3. ラジオモード — アプリがラジオチューナーになる

曲を選ぶことすら面倒な日がありますよね。ラジオモードはアプリ全体をインターネットラジオのチューナーに変えます — 本物の生放送局が何千もあります。

### 入り方 / 出方

プレイヤー画面で **`Alt+Shift+R`** を押します。*「ラジオモードに切り替えますか？」* と聞かれるので確認 — その瞬間、すべてが変わります。色がラジオ専用テーマに変わりますが（わざとです — いまどこにいるか一目で分かるように）、聴いていた音楽のキューはそっくりそのまま保管され、戻ってきたら復元されます。

もう一度 **`Alt+Shift+R`** で通常の音楽モードに戻ります。（一つだけルール: ラジオとローカルデッキは同時に開けません — 片方を出てからもう片方へ。）

### ラジオ局を探す

**`s`** を押して、曲を探すのと同じように検索してください — ただし今度は **Radio Browser**、インターネットラジオ局の巨大な公開リストを検索しています。ジャンル（"jazz"）、国、局名で探して、`Enter` でチューニング。

ラジオモードではライブラリ（`l`）も変わります: **ラジオのお気に入り** と **ラジオ履歴** の二つのタブだけになります — 好きな局と最近聴いた局で、音楽のお気に入りとは完全に別管理です。局の上で **`f`** を押すとお気に入りに。

### 聴いている間

生放送は*生放送*なので、巻き戻しはありません。接続が少し揺れて放送より遅れると、アプリが教えてくれます — *"Live: 25s behind"* — そこで **`r`** を押せばライブの最前線にピタッと戻ります。

いちばんの目玉はこれ: いい曲が流れてきたら **`i`** を押してください。*いま何が流れているか*を教えてくれる小さなカードが出ます — 局が流している放送情報をそのまま読んで。カードの中では:

- **`f`** — 識別された曲を*音楽*のお気に入りに保存（通常モードに戻ればそこにあります）、
- **`g`** — DJ Gem にもっと教えて、と頼んだり似た曲を探してもらったり。

**`Alt+Shift+E`** にはラジオ録音のブラウザもあります。

## 4. ローカルデッキ — 自分のパソコンの音楽を、美しく

ローカルデッキは*あなたのパソコンにある*音楽のための専用プレイヤーです: ダウンロードした曲と、もともと持っていたオーディオファイル。インターネットがまったくなくても動きます。

### 入り方 / 出方

ライブラリ（**`l`**）を開いて、**`Alt+Shift+L`** を押します。*「ローカルプレイヤーモードに切り替えますか？」* と聞かれるので確認 — ローカル音楽の閲覧のためだけに作られた没入型の画面に入ります。もう一度 **`Alt+Shift+L`** で出られます。

### 何が見えるか

ローカルデッキはダウンロードフォルダをスキャンして（*アーティスト / アルバム / 曲* の構造を理解します）、すべてをセクションに整理します。**数字キー**でセクションを行き来してください:

**ホーム · 曲 · アルバム · アーティスト · ジャンル · フォルダ · スマートリスト · スキャンエラー · インポートセッション · 受信箱**

- **曲 / アルバム / アーティスト / ジャンル** — コレクションを好きな切り口で。
- **フォルダ** — ディスク上の置かれ方そのままで閲覧。
- **スマートリスト** — 自動で作られるコレクション。
- **スキャンエラー** — スキャナが読めなかったファイル。黙って消えるものはありません。
- **インポートセッション / 受信箱** — Spotify インポートがレビューを待つ場所（次の章！）。

### 音楽の入れ方

- `d` / `Shift+D` で保存したすべての曲が自動でここに現れます。
- 設定でスキャンするフォルダを追加できます（ローカルデッキのルート）。
- Spotify インポートがここへ直接ダウンロードしてくれることも — 続きをどうぞ。

## 5. Spotify からのお引っ越し — じっくり、ぜんぶ

Spotify のプレイリストといいねした曲を YuTuTui! に持ち込めます。黙って当てずっぽうにされることはありません: すべての曲は実際のタイトル・アーティスト・アルバムでマッチングされ、確信が持てない曲は*あなたが*決められるよう脇に取り分けられます。

**音楽はどこへ行くの？** 選択肢は二つ:

1. **アプリ自身のライブラリプレイリストへ** — すぐ使えて、YouTube アカウント不要。アプリ内インポートはこの方式です。
2. **本物の YouTube Music アカウントへ**（プレイリストやいいね）— コマンドライン方式で、YouTube のサインイン Cookie が必要です（README のリファレンス参照）。

### 5a. 初回のみの設定（約5分、無料）

この設定がなぜあるのか、正直にお話しします: Spotify は自社に登録されたアプリにしかライブラリの読み取りを許可せず、個人向けのアプリ登録（"Development Mode"）はアプリの持ち主が名簿に載せた人しか受け付けません。だからみんなで一つのアプリを共有する代わりに、*各自が自分だけの小さな無料アプリを作る*のです — 5分で終わり、お金はかからず、使うのはあなただけ。秘密のパスワードのようなものはありません; コピーするのは ID 一つだけです。

1. ブラウザで **[developer.spotify.com/dashboard](https://developer.spotify.com/dashboard)** を開き、いつもの Spotify アカウントでログインします。
2. **Create app** をクリック。
3. **App name** と **App description** は何でも — `yututui` で十分です。
4. **Redirect URIs** に正確にこう入力して:

   ```
   http://127.0.0.1:9271/callback
   ```

   **Add** をクリックします。一字一句正確に — `localhost` という単語ではなく数字の `127.0.0.1`（Spotify は localhost を拒否します）、末尾にスラッシュも付けないでください。このアドレスは「このパソコンのアプリに戻ってきて」という意味で、何も外に出ていきません。
5. **Which API/SDKs are you planning to use?** で **Web API** にチェック。
6. 規約に同意して **Save**。
7. 作ったアプリ → **Settings** で **Client ID**（長い英数字の列）をコピーします。「Client secret」は無視してください — 不要です。
8. 同じ設定画面で **User Management** を開き、*自分自身*を追加します: 名前と Spotify アカウントのメールアドレス。これが許可名簿です — この手順を飛ばすと、あとで Spotify が「403」と答えます。

完了。二度とやる必要はありません。

### 5b. YuTuTui! と Spotify をつなぐ

アプリで: **設定（`o`）→ アカウント → Spotify** → Client ID を貼り付け → **Connect**。ブラウザに Spotify の承認ページが開くので承認 — アプリに接続済みと表示されます。

（タイプする方が好きなら `ytt auth spotify --client-id <あなたのID>` でも同じです。ブラウザのないマシンでは、承認リンクがクリップボードにコピーされ `spotify_auth_url.txt` にも保存されるので、どの端末で開いても OK。）

### 5c. アプリの中でインポート

1. **設定 → アカウント → 「Spotify からインポート…」** へ。
2. あなたの Spotify プレイリスト一覧が出ます。一つ選んでください。
3. **インポートモード**を選びます（すぐ横にドロップダウンがあります）:

   | モード | 意味 |
   | --- | --- |
   | **速いインポート (Fast playlist)** | 確実なマッチ*と*安全な近似マッチまで。ほとんどの曲がすぐ入ります。 |
   | **厳格なインポート (Strict playlist)** | アプリが確信できるマッチだけ; 残りはすべてあなたのレビュー待ちに。 |
   | **レビュー優先 (Review first)** | マッチングだけして、まだ何も書き込まない — あとで全部あなたが承認。 |

4. これで終わり — インポートは**音楽が流れたままバックグラウンドで**進みます。ステータス行に進捗が出ます。

終わるとこう表示されます: *「インポート完了 … ライブラリ → プレイリストに保存」*。プレイリストはそこにあり、すぐ再生できます。

### 5d. 残った曲のレビュー

アプリが確信を持てなかった曲は、勝手に埋められることは決してありません — **ローカルデッキ → インポートセッション**（と**受信箱**）で待っています。ローカルデッキ（第4章）へ行き、セッションを開いて行を見てください: Spotify にあった曲と有力候補が並んで表示されます。正しそうなものだけ承認するか — **`Shift+A`** でマッチ済み候補を一括承認もできます。各行からダウンロードの再試行や、候補リンクを開いて自分の耳で確かめることもできます。

### 5e. コマンドライン版 *（任意）*

コマンドに慣れているなら、同じ仕組みをより多くのオプション付きで使えます:

```sh
ytt transfer import <spotify-url-または-id>   # プレイリスト → YTM アカウント (Cookie 必要)
ytt transfer import liked --to likes          # Spotify のいいね → YTM のいいね、順序保持
ytt transfer import <url> --to local:名前      # → アプリ自身のライブラリプレイリスト (YTM 不要)
ytt transfer resume <job-id>                  # 中断したところから再開
ytt transfer backup --dir ~/music-backup      # 全プレイリストをファイルにバックアップ
ytt transfer session <id>                     # インポートセッションの中身を見る
```

インポートにはチェックポイントがあります: レート制限でも、ノートの蓋を閉じても、停電でも — あとで `resume` すれば止まったところから続きます。

### うまくいかないとき

よくあるつまずき（403「許可リスト外」、INVALID_CLIENT、ポート使用中）には、すべて **[README のトラブルシューティング](README.ja.md#トラブルシューティング)** に一行の処方があります。

## 6. 個人データを安全に持ち出す

別のパソコンへ移る前や、自分のライブラリと設定を保管しておきたいときは、YuTuTui! の個人データを一つの読みやすい、バージョン付き JSON ファイルにまとめられます。アプリの中からは:

1. **`o`** で設定を開きます。
2. **一般（General）→ 個人データをエクスポート（Export personal data）** を選び、**`Enter`** を押します。
3. 進行中の表示が消え、保存されたファイルの場所がステータス行に出るまで待ちます。

ファイルは OS の **Downloads** フォルダへ保存されます。設定画面でまだ保存していない変更や、いまの音量・シャッフルなどの状態も、画面を閉じずに現在の内容で取り込まれます。

ターミナルからも同じことができます。既定の Downloads フォルダへ保存するには:

```sh
ytt data export
```

保存先を自分で選ぶには:

```sh
ytt data export --to ~/safe-existing-dir
```

`--to` の保存先は**先に作っておいた実在するディレクトリ**でなければなりません。YuTuTui! は保存先ディレクトリを自動作成せず、既存ファイルも上書きしません。OS の Downloads フォルダが見つからない場合も、カレントディレクトリへ黙って保存せず、`--to DIR` を使うよう案内します。

通常のプライマリ `ytt` またはデーモンが起動中なら、CLI は古い可能性のあるディスクファイルではなく、そのプロセスの最新メモリ状態を受け取ります。旧版、壊れた descriptor、生きているのに到達不能なオーナー、または判定が曖昧な状態では、勝手にディスクへ切り替えずエクスポートを停止します。広告された endpoint が確実に stale で、排他的データロックによりストアのオーナーがいないと確認できた場合だけ、オフラインスナップショットへ安全に復旧します。`--new-instance` プレイヤーも開いている場合、CLI がエクスポートするのは広告されたプライマリだけです。各セカンダリは、その設定画面から個別にエクスポートしてください。オフライン CLI は、セカンダリを含む現行版の ytt オーナーをすべて閉じるまでディスクストアを読みません。

**含まれるもの:**

- 移行可能な設定
- お気に入りと音楽の再生履歴
- ラジオのお気に入りと履歴
- プレイリスト
- パスを含まないローカル曲メタデータと、安全な公開カタログ ID
- 推薦シグナルとステーションの好み

**含まれないもの:**

- Cookie、API キー、OAuth トークン、アカウント識別情報
- ファイルシステムのパス、マシン固有のオーディオデバイス設定
- 再生 URL、取得元 URL、アートワーク URL、ラジオストリーム URL
- 実際のダウンロード曲と録音、ダウンロード manifest、メディア sidecar
- 未送信 scrobble、転送ジョブとレポート、セッションキュー
- AI 使用ログ、生成キャッシュ、アートワークキャッシュ、アプリログ
- 管理対象ツールのバイナリとパス、デスクトップのウィンドウ配置、復旧バックアップ

> **大切な注意:** 認証情報や音楽・録音ファイルそのものは入りませんが、**あなたの再生履歴が入った個人ファイル**です。JSON ファイル自体は暗号化されません。安全な場所に保管し、他人へ送る前に内容を確認してください。

YuTuTui! は既存ファイルを上書きせず、現在のユーザーだけが読める新しいファイルを作ります。信頼できないローカルアカウントが完成パスを作成・置換・削除できる保存先も拒否します。保存先のファイルシステムでこの非公開の権限や ACL を適用・検証できない場合は、広く読めるコピーを残さず、エクスポートを失敗させます。

現在の機能は**エクスポート専用**です。この JSON を `ytt` にインポートしたり、そこから設定やライブラリを復元したりする機能はまだありません。

## 7. うまくいかないとき（どこでも）

いつでも最初の一手: アプリを閉じてこれを実行してください。

```sh
ytt doctor
```

補助プログラムをすべて点検し、何が足りず、どう入手すればいいかを正確に教えてくれます。それ以外のすべて — 再生できない曲、出ないアルバムアート、スクロブル、Spotify のエラー — は **[README のトラブルシューティング表](README.ja.md#トラブルシューティング)** に症状別でまとまっています。

まだ困っていたら？ [issue を立てて](https://github.com/Ochichan/Yututui/issues)、見たままを書いてください — OS も添えて。
