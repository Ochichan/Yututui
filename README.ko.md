# ytm-tui

[English](README.md) · **한국어** · [日本語](README.ja.md)

*터미널 안에서 돌아가는 유튜브 뮤직 플레이어. 버튼 잔뜩 달린 멀쩡한 앱을 보고 "이걸 전부 키보드로 하고, 메모리도 덜 먹고, 카페에서 살짝 우쭐하게 만들면 어떨까?"라고 생각한 사람이 만들었습니다.*

`ytt`(앞으로 입력하게 될 세 글자짜리 명령어입니다)는 유튜브 뮤직 음악을 바로 터미널 창 안에서 재생합니다. 램을 조용히 갉아먹는 브라우저 탭도 없고, 썸네일 모양 광고도 없고, 마우스도 필요 없습니다 — 그리워질까 봐 마우스 지원은 전부 넣어뒀지만요. 그냥 당신, 어두운 화면 위의 글자들, 그리고 솔직히 좀 과하다 싶은 숫자의 키보드 단축키만 있으면 됩니다.

빠르고, 가볍고, 진작 잤어야 할 사람 한 명이 만들었습니다. 그 사람 컴퓨터에서는 잘 돌아갑니다. 이제 당신 컴퓨터에서도 그러길 바랍니다.

## 나한테 맞는 물건일까?

이런 분이라면 십중팔구 맞습니다:

- 어차피 터미널에서 살고 있어서, 브라우저를 여는 게 먼 산책처럼 느껴지는 분.
- 메모리도 거의 안 먹고 광고도 전혀 안 보이는 음악을 원하는 분.
- 키보드 단축키가 일종의 사랑의 언어라고 믿는 분.
- 음악 플레이어를 *완전히* 다시 칠하고, 키를 다시 묶고, 내 취향에 맞게 주무르는 걸 좋아하는 분.

반대로, 커다란 앨범 커버, 전부 마우스로 클릭하는 조작, 매끈한 폰 앱을 원한다면 **안 맞을** 겁니다. 완전히 괜찮아요. 그래도 우리 친구는 할 수 있습니다.

> **스크린샷에 대하여.** 추가하는 중입니다. 지금은 글자와 가끔 등장하는 이모지로 만들어진 음악 플레이어를 상상해 주세요. 아니면 — 이게 더 낫겠네요 — 그냥 설치해서(2분이면 됩니다) 진짜를 보세요.

## 그냥 음악, 지금 당장 (60초 버전)

맥에 [Homebrew](https://brew.sh)가 깔려 있다면? 이 세 줄을 붙여넣으면 끝입니다:

```sh
brew install mpv yt-dlp                 # ytm-tui가 기대는 도우미 두 개
brew install Ochichan/tap/ytm-tui       # ytm-tui 본체
ytt                                      # 실행
```

그다음: `/`를 누르고, 노래 제목을 입력하고, `Enter`, `↑`/`↓`로 이동, 다시 `Enter`로 재생. 끝. 음악이 나옵니다.

윈도우나 리눅스를 쓰거나, Homebrew가 뭔지 모르겠다면? 아래 단계에서 천천히 안내합니다. 그래도 쉽습니다 — 약속해요.

## 제대로 설치하기

`ytm-tui`는 친절한 얼굴 역할입니다. 실제로 "노래를 찾아서 소리를 내는" 일은 사랑받는 무료 프로그램 두 개에게 맡깁니다. 그 둘을 한 번 설치하고, ytm-tui를 설치하고, 실행하면 됩니다. 세 단계예요.

### 1단계 — 도우미 두 개 (mpv + yt-dlp)

- **[mpv](https://mpv.io)** — 실제로 스피커에서 소리가 나오게 하는 엔진.
- **[yt-dlp](https://github.com/yt-dlp/yt-dlp)** — 유튜브에서 음원을 가져오는 녀석.

이 둘은 **딱 한 번만** 설치하면 됩니다. 표에서 내 운영체제를 찾으세요:

| 내 컴퓨터 | 이렇게 입력 |
| --- | --- |
| **macOS** ([Homebrew](https://brew.sh) 사용) | `brew install mpv yt-dlp` |
| **Windows** ([Scoop](https://scoop.sh) 사용) | `scoop install mpv yt-dlp` |
| **Linux** (데비안/우분투) | `sudo apt install mpv yt-dlp` |
| **Linux** (아치) | `sudo pacman -S mpv yt-dlp` |

> **"패키지 매니저"가 뭔가요?** 터미널용 앱스토어 같은 겁니다 — 한 줄로 프로그램을 설치해 주죠. Homebrew(맥), Scoop(윈도우), apt/pacman(리눅스)이 대표적입니다. 내 줄에 있는 게 아직 없다면, 위 링크를 눌러 먼저 설치하세요. 이것도 한 번만 하면 됩니다.

**제대로 됐는지 확인하려면?** `mpv --version`, 그다음 `yt-dlp --version`을 입력해 보세요. "command not found" 대신 글자들이 주르륵 나오면 성공입니다.

### 2단계 — ytm-tui 본체

**쉬운 방법 (추천)** — 패키지 매니저에게 다 맡기기:

```sh
# macOS
brew install Ochichan/tap/ytm-tui

# Windows
scoop bucket add ytm-tui https://github.com/Ochichan/scoop-bucket
scoop install ytm-tui
```

**소스에서 직접 까는 방법** — 이 저장소를 폴더째로 받았다면:

```sh
# macOS / Linux
./install.sh

# Windows (PowerShell)
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

설치 스크립트는 예의가 바릅니다. 당신 컴퓨터에 맞는 완성품이 있으면 그걸 쓰고, 없으면 처음부터 직접 빌드합니다. 처음부터 빌드하려면 [Rust](https://rustup.rs)가 필요하고(먼저 설치하세요), 몇 분 걸리니, 커피 한 잔 내리러 가기 딱 좋은 순간입니다. 설치 폴더를 `PATH`에 슬쩍 추가해서 어디서든 `ytt`를 실행할 수 있게 해주고, mpv와 yt-dlp가 있는지도 다시 확인해 줍니다.

### 3단계 — 실행하기

```sh
ytt
```

세 글자. 이게 명령어 전부입니다.

**가장 먼저 배워야 할 가장 쓸모 있는 것:** 언제든 **`?`**를 누르세요. 모든 키와 그 기능이 적힌, 항상 최신 상태인 전체 목록이 뜹니다. 그래서 이 README 내용을 다 잊어버려도, 앱이 대신 기억해 줍니다.

> **`ytt: command not found`가 뜬다면?** 터미널을 닫고 새 창을 연 뒤 다시 해보세요 — `PATH`가 따라잡을 시간이 필요할 뿐입니다. 그래도 막혔다면 [뭔가 잘못됐을 때](#뭔가-잘못됐을-때)로 가세요.

## 화면 둘러보기

`ytm-tui`는 몇 개의 **화면**으로 이루어져 있습니다. 당신은 항상 그중 정확히 하나를 보고 있고, 키 하나로 그 사이를 오갑니다. 각 화면이 무엇을 위한 것인지, 전체 지도를 펼쳐 보겠습니다.

### 상단 바 (모든 화면 공통)

모든 화면 맨 위에는 이런 모양의 얇은 내비게이션 바가 가로지릅니다:

```
✨ ytm-tui │  플레이어   검색   라이브러리   설정   AI
```

- 맨 왼쪽 **`✨` 반짝이**는 애니메이션 전체 스위치입니다. 클릭하면 모든 움직임이 켜지거나 꺼집니다. (애니메이션이 꺼지면 반짝이는 조용히 사라지지만, 그 자리는 여전히 클릭해서 다시 켤 수 있습니다.)
- **`ytm-tui`**도 클릭할 수 있습니다 — **정보** 카드를 엽니다.
- 구분선 뒤의 다섯 단어 — **플레이어, 검색, 라이브러리, 설정, AI** — 는 탭입니다. 지금 있는 화면이 강조되어 있고, 다른 걸 클릭하면 바로 그 화면으로 점프합니다. (대기열과 가사는 플레이어 *안에* 살기 때문에 별도 탭이 아닙니다.)

모든 화면은 또 흐릿한 **`?  단축키`** 푸터로 끝납니다 — 클릭하거나 `?`를 누르면 전체 치트시트가 뜹니다.

### 플레이어 (본진)

<!-- 📸 screenshot: 플레이어 화면 전체 주석 — 제목, 탐색바, 컨트롤, 상태 글자, 앨범 아트 -->

여기서 대부분의 시간을 보내게 됩니다. 위에서 아래로:

**제목 줄.** 가운데 정렬, 굵게 — 지금 재생 중인 곡을 보여줍니다: `♥ 곡 제목 — 아티스트`. 작은 `♥` 하트는 그 곡이 즐겨찾기일 때만 나타납니다. 아직 아무것도 재생하지 않았다면 **`Nothing playing — press / to search`**(재생 중 없음 — `/`로 검색)라고 적혀 있습니다. (다운로드 완료, 오류 등 알릴 일이 생기면 여기에 잠깐 메시지가 번쩍입니다 — 좋은 소식은 초록, 문제는 빨강.)

**탐색 막대.** 화면 너비만큼 뻗은 진행 막대이고, 시간이 `흐른 시간 / 전체 시간` 형태로 안에 들어 있습니다 — 예를 들어 `2:34 / 4:15`. **막대 아무 곳이나 클릭하면** 그 지점으로 점프합니다.

**컨트롤 버튼.** 가운데에 늘어선 미디어 컨트롤 한 줄:

```
 ⇤     ▸     ⇥       vol  -  50%  +
```

- `⇤` 이전 곡 · `▸` / `‖` 재생-또는-일시정지(일시정지일 때 `▸`, 재생 중일 때 `‖`) · `⇥` 다음 곡. 전부 클릭 가능.
- `vol  -  50%  +`는 볼륨 묶음입니다 — `-`나 `+`를 클릭해 조절하고, 가운데 숫자가 현재 볼륨입니다.

**상태 줄.** 음악이 *어떻게* 재생되고 있는지를 한눈에 알려주는 작은 기호들의 줄입니다. 대부분 클릭할 수 있습니다. 전체 해독표는 이렇습니다:

| 보이는 것 | 의미 | 클릭하면 |
| --- | --- | --- |
| `▸ playing` / `‖ paused` | 현재 재생 상태 | — |
| `1/3` | 대기열에서 3곡 중 1번째 | **대기열**을 엽니다 |
| `🤔` / `👍` / `👎` | 이 곡에 대한 내 평가 (중립 / 좋아요 / 싫어요) | 평가를 순환합니다 |
| `S:🔀` / `S:✗` | 셔플 켜짐 / 꺼짐 | 셔플 토글 |
| `R:✗` / `R:🔁` / `R:🔂` | 반복: 끄기 / 전체 / 한 곡 | 반복 순환 |
| `1.5x` | 재생 속도 (보통 `1.0x`가 아닐 때만 표시) | — |
| `eq:Flat` | 현재 이퀄라이저 프리셋 | **EQ** 메뉴를 엽니다 |
| `norm` | 음량 평준화 켜짐 | — |
| `radio:balanced` | 라디오(자동재생) 켜짐, 그리고 그 모드 | **라디오 모드** 메뉴를 엽니다 |
| `⬇ 45%` / `⬇ ✓` / `⬇ ✗` | 다운로드 진행 중 / 완료 / 실패 | — |

> 이 중 둘은 라벨 바로 아래에 작은 팝업 메뉴를 엽니다: **`eq:`**를 클릭하면 이퀄라이저 프리셋(Flat, Bass, Treble, Vocal, Rock, Jazz)이, **`radio:`**를 클릭하면 세 가지 라디오 분위기(집중, 균형, 발견)가 펼쳐집니다. 하나 고르면 즉시 적용됩니다.

**가운데 큰 영역.** 켜둔 것들로 채워집니다: **앨범 아트**(터미널이 지원하면 진짜 이미지), **가사** 패널(아래 참고), 그리고 화면에 약간의 생기를 원한다면 빈 공간을 떠다니는 잔잔한 [애니메이션](#그래픽-탭)까지.

### 검색

<!-- 📸 screenshot: 검색 화면 — 입력창, 검색 버튼, 하트/길이가 붙은 결과 목록 -->

**`/`**를 눌러 검색합니다. 이렇게 구성됩니다:

- 위쪽에 **검색창**, `Search`라는 라벨이 붙어 있습니다(로그인하지 않았다면 `Search · anonymous` — 정상입니다. 공개 곡은 계정 없이도 잘 재생됩니다). 검색어를 입력하고 `Enter`를 누르거나 옆의 **Search** 버튼을 클릭하세요.
- 그 아래 **결과 목록**. 각 줄은 `♥ 곡 제목 — 아티스트  (3:48)` 형태입니다 — 하트는 이미 즐겨찾기일 때 보이고, 괄호 안 시간은 곡 길이입니다. 선택된 줄은 `▶`로 표시됩니다.
- 검색이 도는 동안에는 **`Searching…`**(검색 중)이 보입니다.

`↑`/`↓`로 이동하고 `Enter`로 선택한 곡을 재생하세요 — 아니면 아무 줄이나 **더블클릭**. 이미 곡이 재생 중이라면, 다른 곡을 고르면 대기열에 추가되고 `Added to queue`(대기열에 추가됨)가 번쩍입니다.

### 라이브러리

<!-- 📸 screenshot: 라이브러리 화면 — 개수가 붙은 네 탭, 트랙 목록, 필터 줄 -->

**`l`**을 누르면 내 개인 컬렉션입니다. 맨 위에 개수가 실시간으로 붙은 네 개의 탭이 있습니다:

```
 전체 (12)   즐겨찾기 (4)   기록 (8)   다운로드 (2)
```

- **전체** — 손댄 모든 것을 한 목록에 모은 것(즐겨찾기이면서 최근 재생한 곡은 한 번만 나옵니다).
- **즐겨찾기** — 하트를 누른 곡들. 추가하려면 앱 어디서든 곡 위에서 `f`를 누르세요.
- **기록** — 최근에 재생한 것, 최신순.
- **다운로드** — 다운로드 폴더에 저장된 음원 파일들, 오프라인 재생 준비 완료.

탭은 클릭하거나 `Tab` / `Shift+Tab`으로 전환합니다. 탭 안에서는 `↑`/`↓`로 이동하고, `Enter`(또는 더블클릭)로 재생합니다.

**필터.** 라이브러리에서 **`/`**를 누르면 현재 탭을 필터링합니다 — 타이핑을 시작하면 목록이 실시간으로 좁혀지고, `[5개]`처럼 일치 개수가 함께 셉니다. `Enter`로 필터를 유지하거나 `Esc`로 지웁니다.

**제거하기.** *전체*를 제외한 모든 탭은 각 줄 끝에 빨간 **`✗`**를 보여줍니다. 클릭하면 그 곡을 제거합니다 — 즐겨찾기에서, 기록에서, 또는 (다운로드 탭에서는) **디스크에서 실제 파일을 삭제**합니다. 마지막 것은 되돌릴 수 없으므로, 다운로드를 지우기 전에 `Delete (Enter) / Cancel (Esc)` 프롬프트로 한 번 더 확인을 받습니다.

> 빈 탭은 스스로 설명합니다. 예: *"즐겨찾기가 아직 없어요 — 곡 위에서 f를 눌러 저장하세요."* 그래서 빈 상자만 보며 뭘 해야 하나 고민할 일이 없습니다.

### 대기열

<!-- 📸 screenshot: 플레이어 위에 뜬 대기열 팝업 — 번호 매겨진 다음 곡 목록, 재생 중 표시 -->

대기열은 재생을 기다리는 곡들의 줄입니다. 플레이어 위에 **팝업**으로 열립니다 — **`c`**를 누르거나, 상태 줄의 `1/3` 위치 카운터를 클릭하세요.

- 각 줄에는 번호가 붙어 있습니다: `▸  1 곡 — 아티스트`. `▸`는 지금 재생 중인 곡을 표시합니다.
- 아무 줄에서나 `Enter`(또는 **더블클릭**)를 누르면 그 곡으로 바로 점프합니다.
- `Delete`를 누르거나 줄의 빨간 **`✗`**를 클릭하면 그 곡을 줄에서 빼냅니다.

`q`를 누르거나 팝업 바깥을 클릭하면 닫힙니다.

### 가사

<!-- 📸 screenshot: 싱크 가사 패널이 스크롤되는 플레이어, 현재 줄 강조 -->

**`L`**(`Shift`+`L`)을 누르면 가사를 보이거나 숨깁니다. 플레이어 가운데 영역, 앨범 아트 아래에 나타납니다.

곡에 **시간 싱크된** 가사가 있으면, 음악에 맞춰 스크롤되고 현재 줄이 가운데에서 강조됩니다 — 1인용 노래방이죠. 불러오는 동안에는 `Fetching lyrics…`(가사 불러오는 중)가, 그 곡에 가사가 없으면 `No synced lyrics found.`(싱크 가사 없음)가 보입니다. (제대로 시간 싱크된 가사만 표시됩니다 — 타임스탬프 없는 일반 가사는 표시되지 않아요.)

### AI 어시스턴트

<!-- 📸 screenshot: AI 어시스턴트 — 대화 기록, 추천 패널, "질문" 입력창, 마스코트 -->

이건 선택 보너스입니다. **`a`**를 누르면 음악 플레이어에게 평범한 말로 말을 걸 수 있습니다. (무료 Google Gemini 키가 필요합니다 — [AI 설정](#ai-탭) 참고 — 하지만 앱의 나머지는 키 없이도 전부 작동합니다.)

화면은 세 부분입니다: 가운데의 **대화** 영역, AI가 곡을 제안할 때 나타나는 **추천 곡** 패널, 그리고 타이핑하는 맨 아래의 **`Ask`(질문)** 칸. 현재 AI 모델 이름은 오른쪽 위 모서리에 있습니다.

원하는 걸 그냥 입력하고 `Enter`를 누르세요:

- *"로파이 음악 좀 틀어줘"*
- *"청소할 때 들을 신나는 곡 세 개 대기열에 넣어줘"*
- *"지금 뭐 나오고 있어?"*
- *"이거 건너뛰고 더 차분한 거 틀어줘"*

내 메시지에는 `you`(나) 태그가, 잠깐의 `…thinking`(생각 중) 뒤에 AI의 답에는 `ai` 태그가 붙습니다. 곡을 제안하면 추천 패널에 `♥ 제목 — 아티스트`로 나타납니다(이미 즐겨찾기면 하트). `Tab`으로 그 목록에 들어가, `↑`/`↓`로 이동하고 `Enter`로 하나 재생하세요.

> 아직 키를 설정하지 않았다면? 화면이 설정 방법과 예시 질문 몇 개로 맞아주고, 첫 메시지를 보낼 때까지 작은 애니메이션 마스코트가 옆을 지켜줍니다. 아무것도 망가지지 않아요. 키를 추가할 때까지 어시스턴트가 잠깐 낮잠을 잘 뿐입니다.

### 왜 이 곡들을? (AI의 선곡 이유)

<!-- 📸 screenshot: "AI 선곡 이유" 오버레이 — 번호 매겨진 곡, 역할 배지, 이유 -->

라디오가 왜 그 곡을 줄 세웠는지 궁금한가요? **`w`**를 누르면 **"AI 선곡 이유"**라는 작은 오버레이가 뜹니다. 다가올 곡들을 나열하고, 각 곡이 이 흐름에서 맡은 역할과 잘 어울린 이유를 짧은 평범한 말로 적어줍니다 — 계산식 없는, 사람이 읽는 "왜". (이번 세션에 AI가 아직 아무것도 줄 세우지 않았다면 오버레이에 보여줄 게 없을 뿐입니다.) `w`나 `Esc`로 닫습니다.

### 정보 & 도움말

- **정보** — **`F1`**(또는 상단 바의 `ytm-tui` 클릭)을 누르면 앱 아이콘, 버전, 라이선스, 제작자, 그리고 GitHub 프로젝트로 가는 클릭 가능한 링크가 담긴 카드가 뜹니다.
- **도움말** — 언제든 **`?`**를 누르면 화면별로 깔끔하게 묶인 전체 단축키 치트시트가 뜹니다. 항상 *당신의* 현재 키를 반영하므로, 무언가를 재설정했다면 기본값이 아니라 당신 버전이 표시됩니다.

아무 패널이나 **닫고** 한 화면 물러서려면 **`q`**. 앱 전체를 **종료**하려면 **`Ctrl+Q`**. 어디서든 플레이어로 점프하려면 **`Ctrl+H`**.

## 설정 — 손잡이 하나하나

**`,`**를 눌러 설정을 엽니다. 위쪽에 **다섯 개의 탭**으로 나뉘어 있습니다: **일반 · 재생 · 핫키 · 그래픽 · AI.**

**조작 방법:**

- `Tab` / `Shift+Tab` — 탭 전환 (또는 클릭).
- `↑` / `↓` — 탭 안의 설정 사이 이동.
- `←` / `→` — 선택한 설정의 값 바꾸기 (토글 켜기/끄기, 슬라이더 밀기, 선택지 순환).
- `Enter` — 토글 뒤집기, 버튼 누르기, 텍스트 칸에 입력 시작.
- `q` — 설정 닫기. 나가는 길에 **자동 저장**됩니다.

> **알아둘 점 하나:** 일부 설정에는 **"(재시작 후 적용)"**이라고 적혀 있습니다. 그건 `ytt`를 껐다 켜야 적용됩니다 — 앱이 무시하는 게 아니라, 실행 중에는 바꿀 수 없을 뿐입니다.

<!-- 📸 screenshot: 설정 화면 — 다섯 탭과 항목 목록 -->

### 일반 탭

매일 쓰는 살림 설정.

| 설정 | 하는 일 |
| --- | --- |
| **언어** | 인터페이스 전체를 **English**와 **한국어** 사이에서 전환합니다. 즉시 바뀝니다. |
| **쿠키 파일** | 로그인하려면 `cookies.txt` 위치를 가리킵니다([쿠키](#쿠키로-로그인하기-선택) 참고). 비워두면 기본 위치를 쓰고, 경로를 입력할 수도 있습니다. 대부분은 건드릴 일이 없습니다. |
| **다운로드 폴더** | 다운로드가 저장되는 폴더. 비우면 기본값(`~/Music/ytm-tui`). 경로를 입력하면 다른 곳으로 보냅니다. |
| **마우스 (재시작 후 적용)** | 마우스 지원을 켜거나 끕니다. 켜짐 = 버튼 클릭, 드래그, 스크롤 가능. 꺼짐 = 순수 키보드. |
| **앨범 아트 (재시작 후 적용)** | 플레이어에 앨범 커버를 진짜 그림으로 보여줍니다. 이미지를 그릴 수 있는 터미널이 필요합니다(많은 터미널이 됩니다). 안 되면 꺼두세요. |
| **앱 시작 시 자동재생** | 켜면 `ytt`가 열리는 순간 멈췄던 자리에서 바로 재생을 이어갑니다. |
| **단축키 초기화** | 버튼입니다(`Enter`). **키보드 단축키만** 기본값으로 되돌립니다 — 재설정하다 막다른 골목에 갇혔을 때 유용해요. |
| **모든 설정 초기화** | 버튼입니다. **전부** — 키, 테마, AI 키까지 — 공장 초기값으로 되돌립니다. 되돌릴 수 없으니 먼저 확인을 받습니다. |

### 재생 탭

두 묶음: **현재 재생**(재생이 어떻게 동작하는지)과 **EQ**(어떻게 들리는지).

**현재 재생**

| 설정 | 하는 일 |
| --- | --- |
| **재생 속도** | `0.5x`(반 속도)부터 `2.0x`(두 배)까지의 슬라이더. `1.0x`가 보통입니다. 까다로운 기타 솔로를 늦추거나 긴 믹스를 빨리 넘길 때 좋아요. |
| **탐색 간격** | `←` / `→` 키가 한 번에 몇 초 점프할지 — 1초부터 60초까지. 취향대로. |
| **갭리스 (재시작 후 적용)** | 곡 사이의 아주 짧은 정적을 없애, 앨범이 끊김 없이 흐르게 합니다. |

**EQ** — 제대로 된 10밴드 이퀄라이저.

| 설정 | 하는 일 |
| --- | --- |
| **프리셋** | 미리 만들어진 음색 고르기: **Flat**(변화 없음), **Bass**, **Treble**, **Vocal**, **Rock**, **Jazz**, 또는 **Custom**. 프리셋을 순환하면 즉시 음색이 바뀝니다. |
| **10개 밴드** (`31 Hz` … `16 kHz`) | 주파수 밴드마다 하나씩, 슬라이더 10개. 각각 **−12 dB ~ +12 dB**. 낮은 쪽을 올리면 더 묵직하게, 높은 쪽을 올리면 더 반짝이게. 어느 밴드든 건드리면 프리셋이 **Custom**으로 바뀌어 당신의 조정이 기억됩니다. |
| **음량 평준화** | 곡 사이의 음량을 고릅니다. 속삭이듯 조용한 곡과 벽을 때리는 곡이 비슷한 크기로 나오죠. 곡이 바뀔 때마다 볼륨 키로 달려갈 일이 없습니다. |

### 핫키 탭

화면별로 묶인 모든 키보드 단축키의 전체 목록. 아무거나 골라 `Enter`를 누른 뒤, 원하는 새 키를 누르면 끝입니다. 그 키가 이미 쓰이고 있으면, 조용히 뭔가를 망가뜨리는 대신 경고로 알려줍니다. 맨 아래에 표시된 초기화 키를 누르면 그 하나의 키만 기본값으로 되돌립니다. (전부 날리고 싶다면? **일반 → 단축키 초기화**가 모두 되돌립니다.)

### 그래픽 탭

`ytm-tui`가 어떻게 보이는지에 관한 모든 것. 세 묶음: **테마**, **색상**, **애니메이션**.

**테마** — 앱 전체를 빠르게 다시 칠하는 법.

| 설정 | 하는 일 |
| --- | --- |
| **프리셋** | 완성된 색 구성표 고르기. 열한 가지: **Default, Midnight, Light, High Contrast, Terminal Green, Gruvbox, Nord, Dracula, Tokyo Night, Solarized Dark, Rosé Pine.** 그중 하나는 당신의 영혼(적어도 다른 터미널 앱들)과 맞을 겁니다. |
| **배경 없음** | 앱 배경을 투명하게 만들어, 터미널 자체의 배경(배경화면, 블러 등)이 비치게 합니다. 반투명 터미널과 잘 어울려요. |

**색상** — 완벽주의자를 위한 칸. 테마 아래에는 인터페이스의 모든 개별 요소가 나열됩니다 — 배경, 여러 단계의 텍스트, 테두리, 강조색, 선택 강조, 탐색바 채움, 플레이어 컨트롤 등. 아무거나 골라 `#ff8800` 같은 **16진수 코드**를 입력하면 *그 부분만* 다시 칠하고, `none`을 입력하면 투명하게 만듭니다. 색 하나하나로 나만의 테마를 처음부터 만드는 방법이죠.

**애니메이션** — `ytm-tui`는 원하는 만큼 생기 넘치게도, 고요하게도 할 수 있습니다. 위 세 스위치가 제어부이고, 나머지는 마음대로 섞을 수 있는 개별 효과입니다.

| **포커스 없을 때 정지** | 터미널 창이 지금 쓰는 창이 아닐 때 모든 애니메이션을 멈춥니다 — ytm-tui가 뒤에 있을 때 전력을 아껴요. |

## 라디오: 멈추지 않는 음악

**`Ctrl+R`**를 누르면 `ytm-tui`는 지금 듣고 있는 곡을 중심으로 한 작은 라디오 방송국이 됩니다. 대기열이 바닥나기 시작하면, 분위기에 어울리는 곡을 조용히 줄 세웁니다 — 그래서 음악이 그냥 멈추질 않아요. 게으른 청취자의 꿈이고, 이건 최고의 칭찬입니다.

얼마나 모험적일지는 **설정 → AI → 라디오 모드**에서(또는 플레이어의 `radio:` 라벨을 클릭해서) 고릅니다:

- **집중** — 집 근처에 머뭅니다. 지금 즐기는 것과 더 비슷한 곡들.
- **균형** — 합리적인 중간. 무난한 기본값.
- **발견** — 길을 벗어나 아직 못 들어본 것들을 찾아 떠납니다.

다음 곡을 *어떻게* 고르는지는 조용히 지켜지는 비밀입니다 — 솔직한 대답이 "엄청난 시행착오와 힘들게 얻은 약간의 자부심"이기 때문이죠. 그냥 잘 됩니다. 특정 선택이 궁금하면 **`w`**를 눌러 평범한 말로 된 이유를 보세요. 아니면, 그냥 즐기세요.

## 다운로드: 오프라인으로 듣기

아무 곡 위에서 **`d`**를 누르면 디스크에 저장되어 인터넷 없이도 들을 수 있습니다. 기본 저장 위치는 여기입니다:

- **macOS:** `~/Music/ytm-tui`
- **Windows:** `%USERPROFILE%\Music\ytm-tui`
- **Linux:** 음악 폴더 안의 `ytm-tui` 폴더

모두 그 폴더 하나에 들어갑니다 — 아티스트별 하위 폴더를 뒤질 필요 없습니다. 다운로드한 곡은 라이브러리의 **전체**와 **다운로드**에 자동으로 나타나, 스트리밍 음악 바로 옆에 자리 잡습니다.

다른 곳에 두고 싶다면? 설정의 **다운로드 폴더**를 바꾸거나, `YTM_DOWNLOAD_DIR` 환경 변수를 설정하세요.

## 리모컨: 어디서든 조종하기

`ytt`가 음악을 재생하는 중이라면, *다른* 터미널에서 — 더 좋게는 키보드의 미디어 키로 — 그 창으로 돌아가지 않고도 조종할 수 있습니다. 마법의 접두사는 **`ytt -r`**입니다 (`-r`은 *remote*, 원격):

```sh
ytt -r pp          # 재생 / 일시정지
ytt -r next        # 다음 곡으로
ytt -r radio on    # 멈추지 않는 라디오 켜기
ytt -r status      # 지금 뭐 나오지?
ytt -r quit        # 음악 멈추고 ytt 종료
```

각 명령은 이미 열려 있는 `ytt`에 조용히 연결해서, 시킨 일 하나를 하고, "이렇게 됐어요" 한 줄을 출력한 뒤 비켜섭니다. 음악 화면은 깜빡이지조차 않습니다.

**전체 명령어** (모두 짧은 별칭이 한두 개씩 있습니다):

| 이렇게 입력하면 | ytt가 하는 일 | 짧은 별칭 |
| --- | --- | --- |
| `ytt -r next` | 다음 곡으로 건너뛰기 | `n` |
| `ytt -r prev` | 이전 곡으로 | `p` |
| `ytt -r play-pause` | 재생 / 일시정지 토글 | `pp`, `toggle` |
| `ytt -r up` / `ytt -r down` | 볼륨 올리기 / 내리기 | `vol-up` / `vol-down` |
| `ytt -r back` / `ytt -r fwd` | 뒤로 / 앞으로 탐색 | `rewind` / `forward` |
| `ytt -r radio [on\|off\|toggle]` | 라디오 켜기·끄기·토글 | — |
| `ytt -r status` | "지금 재생 중" 한 줄 출력 | `st` |
| `ytt -r quit` | 재생 멈추고 앱 종료 | `exit` |

`ytt -r status`는 이런 깔끔한 한 줄로 답합니다:

```
[playing] Bohemian Rhapsody — Queen  •  vol 80%  •  3/12  •  radio on
```

알아두면 좋은 플래그 두 개: **`-q`**를 붙이면 성공 메시지를 숨깁니다(조용해야 하는 미디어 키에 묶을 때 좋아요), **`--json`**을 붙이면 상태 표시줄용으로 기계가 읽기 좋은 JSON을 받습니다. 전체 목록은 `ytt -r --help`로 볼 수 있습니다.

### 미디어 키에 연결하기

여기가 재미있는 부분입니다. 키보드의 작은 `⏯ ⏭ ⏮` 키를 한 번만 가르쳐 두면, 다시는 플레이어 창으로 전환할 일이 없습니다. **i3**나 **sway**라면 설정에 이렇게 넣으세요:

```
bindsym XF86AudioPlay exec ytt -r pp
bindsym XF86AudioNext exec ytt -r next
bindsym XF86AudioPrev exec ytt -r prev
```

키 하나로 명령을 실행할 수 있는 도구라면 무엇이든 똑같이 됩니다 — Hyprland의 `bindl`, GNOME의 사용자 지정 단축키, macOS라면 [skhd](https://github.com/koekeishiya/skhd)나 Karabiner 같은 도구. 그냥 키를 `ytt -r <명령>`에 연결하면 끝입니다.

### 플레이어는 한 번에 하나만

작지만 기분 좋은 부수 효과: `ytt`가 이미 돌고 있으면, `ytt`를 또 입력해도 스피커를 두고 다툴 두 번째 플레이어가 **뜨지 않습니다.** 대신 점잖게 물러납니다:

```
ytt is already running.
  Control it:  ytt -r <command>   (e.g. `ytt -r pp`, `ytt -r next`)
  Stop it:     ytt -r quit
  New player:  ytt --new-instance
```

그래서 무심코 두 번째로 실행해도, 이미 켜둔 녀석에게 말 거는 법을 다시 알려줄 뿐입니다. 그래도 *정말로* 완전히 독립된 두 번째 플레이어를 원한다면, `ytt --new-instance`가 탈출구입니다.

### 스크립트를 짜는 분께

`ytt -r`는 흔한 종료 코드 관례를 따르므로, 스크립트와 키 바인딩 안에서 얌전하게 동작합니다:

- **`0`** — 완료, 다 잘 됨.
- **`1`** — 실행 중인 `ytt`에 닿지 못함(정말 켜져 있나요?).
- **`2`** — 명령이 말이 안 되거나 적용되지 않음(오타 난 명령어, 또는 빈 대기열에서 `next`).

## 쿠키로 로그인하기 (선택)

짧게: **거의 확실히 필요 없습니다.** 로그인하지 않아도 `ytm-tui`는 공개된 곡을 익명으로 잘 검색하고 재생합니다.

로그인이 필요한 경우는, 로그인된 유튜브 뮤직 계정이 *있어야* 들을 수 있는 곡(멤버십 전용 업로드, 일부 지역 제한 곡 등)에 접근할 때뿐입니다. 그러려면 앱에 브라우저의 "쿠키"를 줍니다 — 당신이 당신임을 증명하는 작은 파일이죠.

기본적으로 `cookies.txt`를 여기서 찾습니다:

- **macOS:** `~/Music/ytm-tui/cookies.txt`
- **Windows:** `%USERPROFILE%\Music\ytm-tui\cookies.txt`

그 폴더를 만들고, 브라우저에서 유튜브 / 유튜브 뮤직 쿠키를 **Netscape 형식**의 `cookies.txt`로 내보낸 뒤(브라우저 확장 프로그램으로 할 수 있습니다), 그 파일을 넣고, `ytt`를 다시 시작하세요.

```sh
# macOS / Linux — 폴더 만들기
mkdir -p ~/Music/ytm-tui
```

```powershell
# Windows (PowerShell) — 폴더 만들기
New-Item -ItemType Directory "$HOME\Music\ytm-tui" -Force
```

**설정 → 일반 → 쿠키 파일**에서 다른 파일을 가리킬 수도 있습니다.

> ⚠️ **이 파일은 비밀번호처럼 다루세요.** 로그인된 브라우저 세션처럼 동작할 수 있으니, 공유하지 말고, 저장소에 커밋하지 말고, 어디에도 올리지 마세요.

## 설정은 어디에 저장되나

직접 건드릴 일은 전혀 없지만 — 모든 건 앱 안에서 편집할 수 있습니다 — 궁금한 분을 위해, `ytm-tui`는 설정을 단일 `config.json`에 보관합니다:

- **macOS:** `~/Library/Application Support/ytm-tui/config.json`
- **Linux:** `~/.config/ytm-tui/config.json`
- **Windows:** `%APPDATA%\ytm-tui\config.json`

환경 변수 두 개로 실행 시 설정을 덮어쓸 수 있습니다. 스크립트나 잠깐 시험해 볼 때 유용해요:

- `GEMINI_API_KEY` — AI 키 (설정에 저장된 값보다 우선).
- `YTM_DOWNLOAD_DIR` — 다운로드 위치 (다운로드 폴더 설정보다 우선).

## 뭔가 잘못됐을 때

심호흡하세요. 거의 항상 이 중 하나입니다:

| 증상 | 해결 |
| --- | --- |
| **아무것도 안 나오거나, 재생하자마자 에러가 난다** | mpv나 yt-dlp가 설치 안 됐거나 `PATH`에 없을 겁니다. [1단계](#1단계--도우미-두-개-mpv--yt-dlp)를 다시 하세요. |
| **`ytt: command not found`** | 완전히 새 터미널 창을 여세요. 그래도 그러면 설치 폴더가 `PATH`에 없는 겁니다 — 설치 스크립트가 추가할 정확한 줄을 출력했으니, 그걸 셸 설정 파일에 붙여넣으세요. |
| **특정 곡이 안 나온다** | 일부 곡은 로그인이 필요합니다. [쿠키로 로그인하기](#쿠키로-로그인하기-선택)를 보세요. |
| **어제까진 됐는데 이제 전부 안 나온다** | 유튜브가 수시로 바뀌면서 yt-dlp를 망가뜨립니다. 업데이트하세요: `brew upgrade yt-dlp`(맥), `scoop update yt-dlp`(윈도우), 또는 패키지 매니저(리눅스). |
| **앨범 아트가 안 보인다** | 기본적으로 꺼져 있고 일부 터미널에서만 됩니다. 설정 → 일반에서 **앨범 아트**를 켜고 **재시작**하세요("재시작 후 적용" 설정입니다). |
| **AI 어시스턴트가 응답하지 않는다** | 설정 → AI에 무료 Gemini 키가 있어야 하고, **AI 사용**이 켜져 있어야 합니다. |
| **키를 바꿨더니 모든 게 엉망이 됐다** | 설정 → 일반 → **단축키 초기화**. 다 용서됩니다. |
| **애니메이션이 배터리를 잡아먹는다** | **프레임 레이트**를 낮추거나, **포커스 없을 때 정지**를 켜거나, 그냥 `A`로 전부 끄세요. |

그래도 막혔다면, [GitHub 저장소](https://github.com/Ochichan/ytm-tui)에 이슈를 열고 무슨 일이 있었는지 적어주세요. 운영체제를 알려주면 가산점입니다.
