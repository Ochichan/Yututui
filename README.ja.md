# ytm-tui

[English](README.md) · [한국어](README.ko.md) · **日本語**

*ターミナルの中で動く YouTube Music プレイヤー。ボタンだらけの立派なアプリを見て「これ全部キーボードでやって、メモリも節約して、カフェでちょっと得意げになれたら最高じゃない?」と考えた人が作りました。*

`ytt`(これから打ち込む3文字のコマンドです)は、YouTube Music の音楽をターミナルの中でそのまま再生します。RAM を静かに食い荒らすブラウザタブもなし、サムネイル型の広告もなし、マウスも不要 — 恋しくなったときのためにマウス操作はちゃんと用意してありますが。あなたと、暗い画面の上の文字と、正直ちょっと多すぎるくらいのキーボードショートカットだけ。

速くて、軽くて、本当はもう寝ているべきだった一人の人間が作りました。その人のマシンでは問題なく動きます。今度はあなたのマシンでも、そうなりますように。

## 自分に合っている?

次のどれかに当てはまるなら、たぶん「はい」です:

- もともとターミナルで生活していて、ブラウザを開くのが長い散歩のように感じる。
- メモリをほとんど食わず、広告も一切見えない音楽がほしい。
- キーボードショートカットは一種の愛の言語だと信じている。
- 音楽プレイヤーを*完全に*塗り替え、キーを組み直し、自分好みに作り変えるのが好き。

逆に、大きなアルバムジャケット、すべてマウスでクリックする操作、つやつやのスマホアプリがほしいなら、**合わない**でしょう。それで全然かまいません。それでも友達でいられます。

> **スクリーンショットについて。** 追加中です。今のところは、文字とときどき現れる絵文字でできた音楽プレイヤーを想像してください。あるいは — こちらのほうが早いですが — とにかくインストールして(2分で済みます)本物を見てください。

## とにかく今すぐ音楽を (60秒バージョン)

Mac で [Homebrew](https://brew.sh) が入っているなら? この2行を貼り付ければ完了です:

```sh
brew install Ochichan/tap/ytm-tui       # ytm-tui + ヘルパー(mpv, yt-dlp, ffmpeg)も一緒に
ytt                                      # 起動
```

あとは: `/` を押し、曲名を入力し、`Enter`、`↑`/`↓` で移動、もう一度 `Enter` で再生。以上。音楽が鳴ります。

Windows や Linux の方、Homebrew が何か分からない方は、次のセクションでゆっくり案内します。それでも簡単です — 約束します。

## きちんとインストールする

`ytm-tui` は親しみやすい「顔」の役割です。実際に「曲を探して音を出す」仕事は、愛されている2つの無料プログラムに任せています。その2つを一度入れて、ytm-tui を入れて、起動する。3ステップです。

### ステップ1 — ヘルパー

- **[mpv](https://mpv.io)** — 実際にスピーカーから音を出すエンジン。
- **[yt-dlp](https://github.com/yt-dlp/yt-dlp)** — YouTube から音源を取ってくる係。
- **[ffmpeg](https://ffmpeg.org)** — **ダウンロードのときだけ**必要です(音源をきれいなファイルにまとめてくれます)。ストリーミングだけなら飛ばして OK。

> **ytm-tui 本体をパッケージマネージャーで入れますか(Homebrew・Scoop・Nix・AUR — ステップ2参照)?** それなら、これらのヘルパーも自動で一緒に入るので、そのままステップ2へ進んで構いません。このステップが効いてくるのは、`ytt` バイナリだけを入れる `curl | sh`・ソースビルドの場合です。

ヘルパーを自分で入れるには、表から自分の OS を見つけてください:

| あなたのコンピューター | これを入力 |
| --- | --- |
| **macOS** ([Homebrew](https://brew.sh) を使用) | `brew install mpv yt-dlp ffmpeg` |
| **Windows** ([Scoop](https://scoop.sh) を使用) | `scoop install mpv yt-dlp ffmpeg` |
| **Linux** (Debian/Ubuntu) | `sudo apt install mpv yt-dlp ffmpeg` |
| **Linux** (Arch) | `sudo pacman -S mpv yt-dlp ffmpeg` |

> **「パッケージマネージャー」って何?** ターミナル版のアプリストアのようなものです — 1行でプログラムを入れてくれます。Homebrew(Mac)、Scoop(Windows)、apt/pacman(Linux)が代表的。自分の行にあるものがまだ無ければ、上のリンクから先に入れてください。これも一度きりです。

**うまくいったか確認するには?** `mpv --version`、続いて `yt-dlp --version` と入力してみてください。「command not found」ではなく文字がずらっと出れば成功です。(ffmpeg も含めて一度に確認したい? ステップ2のあとに **`ytt doctor`** を実行してください。)

### ステップ2 — ytm-tui 本体

**簡単な方法(おすすめ)** — 自分のプラットフォーム向けのコマンド1つ。どれも `ytt` だけでなくヘルパー(mpv, yt-dlp, ffmpeg)まで一緒に入れてくれます:

| あなたのコンピューター | コマンド1つ |
| --- | --- |
| **macOS** | `brew install Ochichan/tap/ytm-tui` |
| **Windows** | `scoop bucket add extras; scoop bucket add ytm-tui https://github.com/Ochichan/scoop-bucket; scoop install ytm-tui` |
| **Linux** — どのディストロでも、[Nix](https://nixos.org/download) 使用 | `nix run github:Ochichan/ytm-tui` |
| **Linux** — Arch | `yay -S ytm-tui-bin` |
| **Linux** — その他すべて | `curl -fsSL https://github.com/Ochichan/ytm-tui/releases/latest/download/install.sh \| sh` |

> **Nix** が最も再現性の高い方法です: `nix run github:Ochichan/ytm-tui` は固定された正確なバージョンを mpv/ffmpeg/yt-dlp ごとまとめて実行し、システムの他の部分には触れません。(自分でより新しい `yt-dlp` を `PATH` に置いていれば、そちらが優先されます。)
>
> **`curl | sh`** の1行は、自分の CPU 向けの完成品バイナリをダウンロードし、リリースのチェックサムと照合して検証し、`ytt` を `PATH` に置きます。入るのは `ytt` だけなので、ヘルパーはステップ1で揃えるか — インストール後に `ytt doctor` で何が足りないかを確認してください。

**ソースから自分でビルドする方法** — このリポジトリをクローンして自分でビルドしたい場合:

```sh
# macOS / Linux
./install.sh --build

# Windows (PowerShell)
powershell -ExecutionPolicy Bypass -File .\install.ps1 --build
```

ソースビルドには [Rust](https://rustup.rs) が必要で(先に入れてください)、数分かかるので、コーヒーを淹れに行くのにちょうどいい瞬間です。`--build` なしで `./install.sh` を実行すると完成品バイナリをダウンロードし、自分のプラットフォーム向けの完成品が無いときだけコンパイルに切り替わります。どちらの場合もインストール先フォルダをそっと `PATH` に追加して、どこからでも `ytt` を実行できるようにしてくれます。

### ステップ3 — 起動する

```sh
ytt
```

3文字。これがコマンドのすべてです。

**最初に覚えるべき、いちばん役立つこと:** いつでも **`?`** を押してください。すべてのキーとその機能が載った、常に最新の一覧が出ます。だからこの README の内容を全部忘れても、アプリが代わりに覚えていてくれます。

> **何かおかしい?** **`ytt doctor`** を実行してください — mpv・yt-dlp・ffmpeg が `PATH` にあるか、ytm-tui がファイルを書けるかを点検し、何を直せばいいかを正確に教えてくれます。
>
> **`ytt: command not found` と出る?** ターミナルを閉じて新しいウィンドウを開き、もう一度試してください — `PATH` が追いつくのに少し時間が要るだけです。それでも詰まったら [うまくいかないとき](#うまくいかないとき) へ。

## 画面ツアー

`ytm-tui` はいくつかの**画面**でできています。あなたは常にそのうち1つだけを見ていて、キー1つでその間を行き来します。それぞれの画面が何のためのものか、地図を広げてみましょう。

### トップバー (全画面共通)

どの画面でも、いちばん上に細いナビゲーションバーが横切ります。だいたいこんな形です:

```
✨ ytm-tui │  Player   Search   Library   Settings   AI
```

- 左端の **`✨` きらめき**はアニメーションのマスタースイッチです。クリックするとすべての動きがオン/オフになります。(アニメーションがオフのときは、きらめきは静かに消えますが、その場所は今でもクリックして戻せます。)
- **`ytm-tui`** もクリックできます — **About(情報)** カードを開きます。
- 区切りの後ろの5つの単語 — **Player(プレイヤー)、Search(検索)、Library(ライブラリ)、Settings(設定)、AI** — はタブです。今いる画面が強調され、ほかをクリックするとそこへ直行します。(キューと歌詞はプレイヤーの*中*に住んでいるので、独立したタブではありません。)

どの画面も、薄い **`?  keybindings`** フッターで終わります — クリックするか `?` を押すと全ショートカット表が出ます。

### プレイヤー (拠点)

<!-- 📸 screenshot: プレイヤー画面の全注釈 — タイトル、シークバー、コントロール、ステータス記号、アルバムアート -->

ここで大半の時間を過ごします。上から下へ:

**タイトル行。** 中央揃え・太字で、再生中の曲を表示します: `♥ 曲のタイトル — アーティスト`。小さな `♥` ハートは、その曲がお気に入りのときだけ現れます。まだ何も再生していなければ **`Nothing playing — press / to search`**(再生中なし — `/` で検索)と書かれています。(ダウンロード完了やエラーなど知らせることがあると、ここに短いメッセージが点滅します — 良い知らせは緑、トラブルは赤。)

**シークバー。** 画面幅いっぱいの進捗バーで、時間が `経過 / 全体` の形で中に入っています — 例えば `2:34 / 4:15`。**バーのどこでもクリックすると**その位置にジャンプします。

**コントロールボタン。** 中央に並ぶメディアコントロールの行:

```
 ⇤     ▸     ⇥       vol  -  50%  +
```

- `⇤` 前の曲 · `▸` / `‖` 再生/一時停止(一時停止中は `▸`、再生中は `‖`)· `⇥` 次の曲。すべてクリック可。
- `vol  -  50%  +` は音量のかたまり — `-` や `+` をクリックして調整、真ん中の数字が現在の音量です。

**ステータス行。** 音楽が*どのように*再生されているかを一目で伝える、小さな記号の列です。ほとんどがクリックできます。完全な解読表はこちら:

| 表示 | 意味 | クリックすると |
| --- | --- | --- |
| `▸ playing` / `‖ paused` | 現在の再生状態 | — |
| `1/3` | キューの3曲中1曲目 | **キュー**を開く |
| `🤔` / `👍` / `👎` | この曲への評価(中立 / 高評価 / 低評価) | 評価を切り替える |
| `S:🔀` / `S:✗` | シャッフル オン / オフ | シャッフル切り替え |
| `R:✗` / `R:🔁` / `R:🔂` | リピート: オフ / 全体 / 1曲 | リピート切り替え |
| `1.5x` | 再生速度(通常の `1.0x` でないときだけ表示) | — |
| `eq:Flat` | 有効なイコライザープリセット | **EQ** メニューを開く |
| `norm` | ラウドネス正規化がオン | — |
| `radio:balanced` | ラジオ(自動再生)がオン、およびそのモード | **ラジオモード**メニューを開く |
| `⬇ 45%` / `⬇ ✓` / `⬇ ✗` | ダウンロード進行中 / 完了 / 失敗 | — |

> このうち2つはラベルのすぐ下に小さなポップアップメニューを開きます: **`eq:`** をクリックするとイコライザープリセット(Flat, Bass, Treble, Vocal, Rock, Jazz)が、**`radio:`** をクリックすると3つのラジオの雰囲気(Focused, Balanced, Discovery)が出てきます。1つ選べば即座に適用されます。

**真ん中の大きな領域。** オンにしたもので埋まります: **アルバムアート**(ターミナルが対応していれば本物の画像)、**歌詞**パネル(下記参照)、そして画面に少し命がほしければ、空きスペースを漂う穏やかな[アニメーション](#graphics-テーマ色アニメーション)まで。

### 検索

<!-- 📸 screenshot: 検索画面 — 入力欄、検索ボタン、ハート/長さ付きの結果リスト -->

**`/`** を押して検索します。構成はこう:

- 上部に**検索ボックス**、`Search` というラベル付き(未ログインなら `Search · anonymous` — 正常です。公開曲はアカウント無しでも再生できます)。左側には **⌕** の虫めがねアイコンが付いています。クエリを入力して `Enter`、または **⌕** アイコンや横の **Search** ボタンをクリック。
- その下に**結果リスト**。各行は `♥ 曲のタイトル — アーティスト  (3:48)`。ハートはすでにお気に入りのとき、括弧内の時間は曲の長さです。選択中の行は `▶` で示されます。
- 検索中は **`Searching…`**(検索中)が表示されます。

`↑`/`↓` で移動し、`Enter` で選択した曲をその場で再生します — 現在のキューは保たれるので、並べてある曲はその後ろに続きます。代わりに `\`(または行を**右クリック**)を押すと、再生中の曲を止めずにキューへ追加します(`Added to queue`(キューに追加)が点滅して教えてくれます)。どの行でも**ダブルクリック**で再生できます。

### ライブラリ

<!-- 📸 screenshot: ライブラリ画面 — 件数付きの4タブ、トラックリスト、フィルター行 -->

**`l`** を押すと自分のコレクションです。上部に、件数がリアルタイムで付く4つのタブがあります:

```
 All (12)   Favorites (4)   History (8)   Downloads (2)
```

- **All(すべて)** — 触れたものすべてを1つのリストにまとめたもの(お気に入りかつ最近再生した曲は一度だけ出ます)。
- **Favorites(お気に入り)** — ハートを付けた曲。追加するにはアプリのどこでも曲の上で `f` を押します。
- **History(履歴)** — 最近再生したもの、新しい順。
- **Downloads(ダウンロード)** — ダウンロードフォルダに保存された音源ファイル。オフライン再生の準備完了。

タブはクリック、または `Tab` / `Shift+Tab` で切り替えます。タブ内では `↑`/`↓` で移動、`Enter`(またはダブルクリック)で選択した曲をその場で再生します — 現在のキューは保たれます。代わりに `\`(または行を**右クリック**)でキューに追加、または `P`(`Shift`+`P`)で現在のタブ全体を新しいキューとして再生できます。

**フィルター。** ライブラリで **`/`** を押すと現在のタブを絞り込みます — 入力を始めるとリストがリアルタイムで狭まり、`[5 matches]` のように一致件数も数えます。`Enter` でフィルターを保持、`Esc` でクリア。

**削除する。** *All* 以外のすべてのタブは、各行の末尾に赤い **`✗`** を表示します。クリックするとその曲を削除します — お気に入りから、履歴から、または(Downloads タブでは)**ディスクから実際のファイルを削除**します。最後のものは取り消せないので、ダウンロードを消す前に `Delete (Enter) / Cancel (Esc)` の確認が出ます。

> 空のタブはそれ自身を説明します。例: *"No favorites yet — press f on a track to save it."*(お気に入りはまだありません — 曲の上で f を押して保存)。だから空っぽの箱を見て何をすべきか悩むことがありません。

### キュー

<!-- 📸 screenshot: プレイヤー上に出たキューのポップアップ — 番号付きの次曲リスト、再生中マーク -->

キューは再生待ちの曲の列です。プレイヤーの上に**ポップアップ**で開きます — **`c`** を押すか、ステータス行の `1/3` という位置カウンターをクリック。

- 各行には番号が付いています: `▸  1 曲 — アーティスト`。`▸` は今まさに再生中の曲を示します。
- どの行でも `Enter`(または**ダブルクリック**)を押すと、その曲へ直接ジャンプします。
- `Delete` を押すか、行の赤い **`✗`** をクリックすると、その曲を列から外します。

`q` を押すか、ポップアップの外をクリックすると閉じます。

### 歌詞

<!-- 📸 screenshot: 同期歌詞パネルがスクロールするプレイヤー、現在行を強調 -->

**`L`**(`Shift`+`L`)を押すと歌詞を表示/非表示します。プレイヤーの真ん中の領域、アルバムアートの下に現れます。

曲に**時間同期された**歌詞があれば、音楽に合わせてスクロールし、現在の行が中央で強調されます — 一人カラオケですね。読み込み中は `Fetching lyrics…`(歌詞取得中)、その曲に歌詞が無ければ `No synced lyrics found.`(同期歌詞なし)と出ます。(きちんと時間同期された歌詞のみ表示します — タイムスタンプの無い普通の歌詞は表示されません。)

### AI アシスタント

<!-- 📸 screenshot: AI アシスタント — 会話ログ、提案パネル、"Ask" 入力欄、マスコット -->

これは任意のおまけです。**`a`** を押すと、音楽プレイヤーに普通の言葉で話しかけられます。(無料の Google Gemini キーが必要です — [AI 設定](#ai-アシスタント--ラジオ) 参照 — でもアプリの残りはキー無しでも全部動きます。)

画面は3つの部分です: 真ん中の**会話**エリア、AI が曲を提案したときに現れる**提案 (Suggestions)** パネル、そして入力する最下部の **`Ask`(質問)** 欄。現在の AI モデル名は右上の角にあります。

ほしいものをそのまま入力して `Enter` を押すだけ:

- *"play some lo-fi beats"*(ローファイを流して)
- *"queue three upbeat songs for cleaning"*(掃除用にアップな曲を3つキューに)
- *"what's playing right now?"*(今かかってるのは?)
- *"skip this and play something calmer"*(これを飛ばして、もっと穏やかなのを)

自分のメッセージには `you`(あなた)タグ、短い `…thinking`(考え中)の後、AI の返答には `ai` タグが付きます。曲を提案すると、提案パネルに `♥ タイトル — アーティスト` として現れます(すでにお気に入りならハート)。`Tab` でそのリストに入り、`↑`/`↓` で移動、`Enter` で1曲再生します。

> まだキーを設定していない? 画面が設定方法と例文いくつかで出迎えてくれて、最初のメッセージを送るまで小さなアニメーションのマスコットが寄り添ってくれます。何も壊れません。キーを足すまでアシスタントがちょっと昼寝するだけです。

### なぜこの曲を? (AI の選曲理由)

<!-- 📸 screenshot: "Why these AI picks" オーバーレイ — 番号付きの曲、役割バッジ、理由 -->

ラジオがなぜその曲を並べたのか気になる? **`w`** を押すと **"Why these AI picks"**(AI の選曲理由)という小さなオーバーレイが出ます。これから来る曲を並べ、それぞれがこの流れで担う役割と、しっくり来た理由を短い普通の言葉で書いてくれます — 計算式抜きの、人が読む「なぜ」。(今回のセッションで AI がまだ何も並べていなければ、オーバーレイには見せるものがないだけです。)`w` か `Esc` で閉じます。

### 情報 & ヘルプ

- **About(情報)** — **`F1`**(またはトップバーの `ytm-tui` をクリック)で、アプリのアイコン、バージョン、ライセンス、作者、そして GitHub プロジェクトへのクリック可能なリンクが載ったカードが出ます。
- **Help(ヘルプ)** — いつでも **`?`** で、画面ごとにきれいにまとめられた全ショートカット表が出ます。常に*あなたの*現在のキーを反映するので、何かを割り当て直していれば、初期値ではなくあなたの版が表示されます。

どのパネルでも**閉じて**1画面戻るには **`q`**。アプリ全体を**終了**するには **`Ctrl+Q`**。どこからでもプレイヤーへ飛ぶには **`Ctrl+H`**。

## 設定 — つまみを一つずつ

**`,`** を押して設定を開きます。上部に**5つのタブ**で分かれています: **General · Playback · Hotkeys · Graphics · AI.**

**操作方法:**

- `Tab` / `Shift+Tab` — タブ切り替え(またはクリック)。
- `↑` / `↓` — タブ内の設定間を移動。
- `←` / `→` — 選択中の設定の値を変える(トグルのオン/オフ、スライダー移動、選択肢の巡回)。
- `Enter` — トグルを反転、ボタンを押す、テキスト欄に入力開始。
- `q` — 設定を閉じる。出るときに**自動保存**されます。

> **覚えておくこと:** 一部の設定には **"(next launch)"**(再起動後に反映)と書かれています。それは `ytt` を一度終了して開き直すと反映されます — アプリが無視しているわけではなく、起動中は変えられないだけです。

### Playback (再生・速度・EQ)

2つのグループ: **Now Playing**(再生の挙動)と **EQ**(音作り)。

**Now Playing**

| 設定 | はたらき |
| --- | --- |
| **Playback speed** | `0.5x`(半速)から `2.0x`(2倍)までのスライダー。`1.0x` が普通です。難しいギターソロを遅くしたり、長いミックスを早送りしたりに最適。 |
| **Seek interval** | `←` / `→` キーが1回で何秒ジャンプするか — 1秒から60秒まで。お好みで。 |
| **Gapless** *(next launch)* | 曲間のごく短い無音を取り除き、アルバムが途切れず流れるようにします。 |

**EQ** — ちゃんとした10バンドイコライザー。

| 設定 | はたらき |
| --- | --- |
| **Preset** | 出来合いの音色を選ぶ: **Flat**(変化なし)、**Bass**、**Treble**、**Vocal**、**Rock**、**Jazz**、または **Custom**。プリセットを巡回すると即座に音色が変わります。 |
| **10本のバンド** (`31 Hz` … `16 kHz`) | 周波数帯ごとに1本ずつ、スライダー10本。各 **−12 dB ～ +12 dB**。低い側を上げれば重く、高い側を上げればきらびやかに。どれかを触るとプリセットが **Custom** に変わり、あなたの調整が記憶されます。 |
| **Normalize (loudness)** | 曲間の音量をそろえます。ささやくほど静かな曲と、壁のように分厚い曲が、ほぼ同じ大きさで鳴ります。曲が変わるたびに音量キーへ走らずに済みます。 |

## ラジオ: 止まらない音楽

**`Ctrl+R`** を押すと、`ytm-tui` は今聴いている曲を中心にした小さなラジオ局になります。キューが尽きかけると、雰囲気に合う曲を静かに並べていきます — だから音楽がただ止まらないのです。ぐうたらなリスナーの夢で、これは最高の褒め言葉です。

どれだけ冒険的かは **Settings → AI → Radio mode** で(またはプレイヤーの `radio:` ラベルをクリックして)選べます:

- **Focused** — 近場にとどまります。今楽しんでいるものにもっと近い曲。
- **Balanced** — 理にかなった中間。無難な既定値。
- **Discovery** — 道を外れて、まだ聴いたことのないものを探しに行きます。

次の曲を*どうやって*選ぶかは、そっと守られた秘密です — 正直な答えが「膨大な試行錯誤と、苦労して得たちょっぴりの誇り」だからです。とにかくうまくいきます。特定の選曲が気になったら **`w`** を押して普通の言葉の理由を見てください。そうでなければ、ただ楽しんで。

## ダウンロード: オフラインで聴く

どの曲でも上で **`d`** を押すと、ディスクに保存され、インターネット無しでも聴けます。既定の保存先はこちら:

- **macOS:** `~/Music/ytm-tui`
- **Windows:** `%USERPROFILE%\Music\ytm-tui`
- **Linux:** Music フォルダ内の `ytm-tui` フォルダ

すべてその1つのフォルダに入ります — アーティストごとのサブフォルダを掘る必要はありません。ダウンロードした曲はライブラリの **All** と **Downloads** に自動で現れ、ストリーミングの音楽のすぐ隣に並びます。

別の場所に置きたい? 設定の **Download dir** を変えるか、`YTM_DOWNLOAD_DIR` 環境変数を設定してください。

## リモコン: どこからでも操作する

`ytt` が再生中なら、*別の*ターミナルから — もっと良いのはキーボードのメディアキーから — その画面に戻ることなく操作できます。魔法の接頭辞は **`ytt -r`**(`-r` は *remote*、リモート):

```sh
ytt -r pp          # 再生 / 一時停止
ytt -r next        # 次の曲へ
ytt -r radio on    # 止まらないラジオをオン
ytt -r status      # 今かかってるのは?
ytt -r quit        # 音楽を止めて ytt を終了
```

どれも、すでに開いている `ytt` にそっと接続し、頼んだ一つのことをして、「こうなりました」と一行表示し、すっと退きます。音楽画面はちらつきもしません。

**コマンド一覧**(どれも短い別名が一つ二つあります):

| こう入力すると | ytt はこうする | 別名 |
| --- | --- | --- |
| `ytt -r next` | 次の曲へスキップ | `n` |
| `ytt -r prev` | 前の曲へ戻る | `p` |
| `ytt -r play-pause` | 再生 / 一時停止の切り替え | `pp`, `toggle` |
| `ytt -r up` / `ytt -r down` | 音量を上げる / 下げる | `vol-up` / `vol-down` |
| `ytt -r back` / `ytt -r fwd` | 後ろ / 前へシーク | `rewind` / `forward` |
| `ytt -r radio [on\|off\|toggle]` | ラジオをオン・オフ・切り替え | — |
| `ytt -r status` | 「再生中」の一行を表示 | `st` |
| `ytt -r quit` | 再生を止めてアプリを終了 | `exit` |

`ytt -r status` はこんな整った一行で答えます:

```
[playing] Bohemian Rhapsody — Queen  •  vol 80%  •  3/12  •  radio on
```

覚えておくと便利なフラグが二つ: **`-q`** を付けると成功メッセージを抑えます(メディアキーに割り当てて静かにさせたいときに)、**`--json`** を付けるとステータスバー向けに機械可読な JSON を受け取れます。全体は `ytt -r --help` で確認できます。

### メディアキーに割り当てる

ここが楽しいところ。キーボードの小さな `⏯ ⏭ ⏮` キーに一度教えておけば、もうプレイヤーの画面に切り替える必要はありません。**i3** や **sway** なら、設定にこう書き加えます:

```
bindsym XF86AudioPlay exec ytt -r pp
bindsym XF86AudioNext exec ytt -r next
bindsym XF86AudioPrev exec ytt -r prev
```

キー一つでコマンドを実行できるツールなら、どれも同じ要領です — Hyprland の `bindl`、GNOME のカスタムショートカット、macOS なら [skhd](https://github.com/koekeishiya/skhd) や Karabiner といったツール。キーを `ytt -r <コマンド>` に向けるだけ。

### プレイヤーは一度に一つだけ

小さいけれど嬉しい副作用: `ytt` がすでに動いているとき、もう一度 `ytt` と入力しても、スピーカーを奪い合う二つ目のプレイヤーは**起動しません。** 代わりに行儀よく身を引きます:

```
ytt is already running.
  Control it:  ytt -r <command>   (e.g. `ytt -r pp`, `ytt -r next`)
  Stop it:     ytt -r quit
  New player:  ytt --new-instance
```

だからうっかり二度目を起動しても、すでに開いている方への話しかけ方を教えてくれるだけ。それでも*どうしても*完全に独立した二つ目のプレイヤーが欲しいなら、`ytt --new-instance` が抜け道です。

### スクリプトを書く人へ

`ytt -r` は一般的な終了コードの作法に従うので、スクリプトやキーバインドの中でも行儀よく振る舞います:

- **`0`** — 完了、すべて良好。
- **`1`** — 実行中の `ytt` に届かなかった(本当に起動していますか?)。
- **`2`** — コマンドが意味をなさない、または適用できない(打ち間違えた動詞や、空のキューでの `next`)。

## Cookie でログインする (任意)

短く言うと: **ほぼ確実に不要です。** ログインしなくても、`ytm-tui` は公開曲を匿名でちゃんと検索・再生します。

ログインが要るのは、ログイン済みの YouTube Music アカウントが*必要な*曲(メンバー限定アップロード、一部の地域制限曲など)にアクセスするときだけ。そのためにはアプリにブラウザの「Cookie」を渡します — あなたがあなたであることを証明する小さなファイルです。

既定では `cookies.txt` をここで探します:

- **macOS:** `~/Music/ytm-tui/cookies.txt`
- **Windows:** `%USERPROFILE%\Music\ytm-tui\cookies.txt`

そのフォルダを作り、ブラウザから YouTube / YouTube Music の Cookie を **Netscape 形式**の `cookies.txt` として書き出し(ブラウザ拡張でできます)、そのファイルを入れて、`ytt` を再起動してください。

```sh
# macOS / Linux — フォルダを作る
mkdir -p ~/Music/ytm-tui
```

```powershell
# Windows (PowerShell) — フォルダを作る
New-Item -ItemType Directory "$HOME\Music\ytm-tui" -Force
```

**Settings → General → Cookies file** で別のファイルを指すこともできます。

> ⚠️ **このファイルはパスワードのように扱ってください。** ログイン済みのブラウザセッションのように働くので、共有せず、リポジトリにコミットせず、どこにも投稿しないこと。

## 設定はどこに保存される?

直接触る必要は一切ありません — すべてアプリ内で編集できます — が、気になる方へ、`ytm-tui` は設定を単一の `config.json` に保管します:

- **macOS:** `~/Library/Application Support/ytm-tui/config.json`
- **Linux:** `~/.config/ytm-tui/config.json`
- **Windows:** `%APPDATA%\ytm-tui\config.json`

2つの環境変数で起動時に設定を上書きできます。スクリプトやお試しに便利です:

- `GEMINI_API_KEY` — AI キー(設定に保存された値より優先)。
- `YTM_DOWNLOAD_DIR` — ダウンロード先(Download dir 設定より優先)。

## うまくいかないとき

深呼吸して。ほぼ必ずこのどれかです:

| 症状 | 直し方 |
| --- | --- |
| **何も鳴らない、または再生した瞬間にエラー** | mpv か yt-dlp が入っていないか、`PATH` に無いはず。`ytt doctor` で何が足りないか特定するか、[ステップ1](#ステップ1--ヘルパー) をやり直してください。 |
| **`ytt: command not found`** | まっさらな新しいターミナルウィンドウを開いてください。それでも続くなら、インストール先が `PATH` に無いということ — インストーラーが追加すべき正確な行を出力したので、それをシェルの設定ファイルに貼り付けてください。 |
| **特定の曲が鳴らない** | 一部の曲はログインが必要です。[Cookie でログインする](#cookie-でログインする-任意) を見てください。 |
| **昨日は動いたのに今日は全部鳴らない** | YouTube が頻繁に変わり、yt-dlp を壊します。更新しましょう: `brew upgrade yt-dlp`(Mac)、`scoop update yt-dlp`(Windows)、またはパッケージマネージャー(Linux)。 |
| **アルバムアートが出ない** | 既定でオフ、かつ一部のターミナルだけ対応です。Settings → General で **Album art** をオンにして**再起動**してください("next launch" 設定です)。 |
| **AI アシスタントが応答しない** | Settings → AI に無料の Gemini キーがあり、**Enable AI** がオンになっている必要があります。 |
| **キーを変えたら全部めちゃくちゃに** | Settings → General → **Reset keybindings**。すべて水に流します。 |
| **アニメーションがバッテリーを食う** | **Frame rate** を下げる、**Pause when unfocused** をオンにする、あるいは単に `A` で全部オフ。 |

それでも詰まったら、[GitHub リポジトリ](https://github.com/Ochichan/ytm-tui) に Issue を立てて、何が起きたか書いてください。OS を書いてくれると加点です。
