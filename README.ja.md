# ytm-tui

[English](README.md) · [한국어](README.ko.md) · **日本語**

*ターミナルの中で動く YouTube Music プレイヤー。ボタン付きの立派なアプリを見て、「これ全部キーボードで操作できて、メモリも食わなくて、カフェでちょっと得意げになれたらどうだろう？」と考えた人が作りました。*

`ytt`（これからあなたが打ち込むコマンドです）は、YouTube Music の音楽をターミナルのウィンドウの中で直接再生します。RAM をこっそり食い荒らすブラウザのタブもなし、サムネイル型の広告もなし、マウスも不要 — 恋しくなったとき用にマウス対応は入れてありますが。あるのはあなたと、黒い画面の上の文字と、正直ちょっと多すぎるキーボードショートカットだけです。

速くて、軽くて、本当はもう寝ているべきだった一人の人間が作りました。その人のマシンでは動きます。そして願わくは、あなたのマシンでも。

## 自分に合ってる？

たぶん！ こんな人なら `ytm-tui` を気に入る可能性が高いです：

- どうせターミナルに住んでいて、ブラウザに切り替えるのが遠い散歩のように感じる人。
- メモリをほとんど使わず、広告で散らからない音楽が欲しい人。
- キーボードショートカットは一種の愛の言語だと信じている人。

逆に、大きなアルバムジャケット、何でもマウスで操作、洗練されたスマホアプリが欲しいなら、**合わない**でしょう。大丈夫です。それでも友達ではいられます。

> **スクリーンショットについて。** ありません。音楽プレイヤーを思い浮かべて、それが全部文字でできていると想像してください。あるいは — こっちの方が早いですが — とりあえずインストールして（2分ほどです）、私の下手なトリミングではなく本物を見てください。

## 「とにかく音楽。今すぐ。」（30秒版）

Mac を使っていて、すでに [Homebrew](https://brew.sh) が入っているなら：

```sh
brew install mpv yt-dlp                 # ytm-tui に必要な2つの相棒
brew install Ochichan/tap/ytm-tui       # ytm-tui 本体
ytt                                      # 実行
```

そのあと `/` を押して、曲名を入力して、`Enter` を押し、`↑`/`↓` + `Enter` で1つ選べば完了。音楽が流れます。

それ以外のみなさん（Windows、Linux、「Homebrew って何？」）は、続きを読んでください — それでも簡単です、約束します。

## ステップ1 — 2つの相棒をインストール

`ytm-tui` は車輪の再発明をしません。実際に「音を見つけて再生する」仕事は、愛されている2つの無料プログラムにそっと任せています：

- **[mpv](https://mpv.io)** — スピーカーから音を出してくれる子。
- **[yt-dlp](https://github.com/yt-dlp/yt-dlp)** — YouTube から音源を取ってきてくれる子。

この2つは**一度だけ**インストールすれば OK です。お使いの OS を選んでください：

| あなたのパソコン | これを入力 |
| --- | --- |
| **macOS**（[Homebrew](https://brew.sh) を使用） | `brew install mpv yt-dlp` |
| **Windows**（[Scoop](https://scoop.sh) を使用） | `scoop install mpv yt-dlp` |
| **Linux**（Debian/Ubuntu） | `sudo apt install mpv yt-dlp` |
| **Linux**（Arch） | `sudo pacman -S mpv yt-dlp` |

> **「パッケージマネージャ」って何？** ターミナル用のアプリストアのようなものです。一行でプログラムをインストールしてくれます。Homebrew（Mac）、Scoop（Windows）、apt/pacman（Linux）が代表的です。表にあるものがまだ無ければ、リンクをクリックして先にインストールしてください — これも一度きりです。

ちゃんと入ったか確認するには？ `mpv --version` と `yt-dlp --version` を入力してみてください。「command not found」ではなく文字がずらずら出れば成功です。

## ステップ2 — ytm-tui をインストール

**簡単な方法**（おすすめ） — パッケージマネージャに全部任せる：

```sh
# macOS
brew install Ochichan/tap/ytm-tui

# Windows
scoop bucket add ytm-tui https://github.com/Ochichan/scoop-bucket
scoop install ytm-tui
```

**ソースから入れる方法** — このリポジトリをフォルダごと手に入れた場合：

```sh
# macOS / Linux
./install.sh

# Windows (PowerShell)
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

インストーラは礼儀正しいです。あなたのパソコン向けの完成品があればそれを使い、無ければ一から自分でビルドします。一からビルドするには [Rust](https://rustup.rs) が必要で（先に入れてください）、数分かかるので、コーヒーを淹れに行くのにちょうどいい瞬間です。インストール先を `PATH` にそっと追加して、どこからでも `ytt` を実行できるようにし、mpv と yt-dlp があるかも再確認してくれます。

## ステップ3 — 実行する

```sh
ytt
```

これだけ。3文字です。

まず最初に覚えること：いつでも **`?`** を押してください。すべてのキーとその機能が書かれた、常に最新の一覧が表示されます — 下の内容を全部忘れても、アプリが代わりに覚えていてくれます。

> **「`ytt: command not found`」と出る？** ターミナルを閉じて新しいウィンドウを開き、もう一度試してください — `PATH` が追いつくのに少し時間が要ります。それでもダメなら [うまくいかないとき](#うまくいかないとき) を見てください。

## 実際の使い方

`ytm-tui` はいくつかの**画面**でできていて、その間を行き来します。あなたは常にそのどれか1つにいます。全体のマップはこうです：

- **🏠 プレイヤー** — 本拠地です。再生中の曲、進行バー、音量、メディアボタンの列が表示されます。ほとんどすべてここから始まります。
- **🔍 検索** — `/` を押して、欲しいものを入力して、`Enter`。`↑`/`↓` で結果を移動し、`Enter` で選んだ曲を再生します。
- **📚 ライブラリ** — `l` を押します。お気に入りの曲とダウンロードしたファイルが一か所に。`All`、`Downloads`、そしてお気に入りを見て回れます。
- **📜 キュー（再生待ち）** — `c` を押します。次に流れる曲の列です。`Enter` で特定の曲へ一気にジャンプ、`Delete` で列から外します。
- **🎤 歌詞** — `L`（Shift+L）を押します。曲に同期歌詞があれば、音楽に合わせてスクロールします。一人カラオケですね。
- **⚙️ 設定** — `,`（カンマ）を押します。すべてのつまみとダイヤルがここに。[自分好みにする](#自分好みにする設定) を見てください。
- **🤖 AI アシスタント** — `a` を押します。普通の言葉で話しかけられます（任意 — 無料のキーが必要。下記参照）。

どのパネルも**閉じて**一歩下がるには **`q`**。アプリ全体を**終了**するには **`Ctrl+Q`**。どこからでもプレイヤーへ瞬間移動するには **`Ctrl+H`**。

ふつうの初回はこんな流れです：

1. `/` を押して曲を検索。→ 2. `Enter` で再生。→ 3. 気に入った曲に `f` で ♥ お気に入り。→ 4. `Ctrl+R` でラジオをオンにして音楽が永遠に続くように。→ 5. `,` で設定に入ってきれいに飾る。

## キー操作

覚えなくて大丈夫です。アプリで `?` を押せばチートシートがそこにあります。それでも気になる人のために、日常的によく使うものを。

### プレイヤーで（本拠地）

| キー | 機能 |
| --- | --- |
| `Space` | 再生 / 一時停止 |
| `←` / `→` | 数秒戻る / 進む |
| `↑` / `↓` | 音量アップ / ダウン |
| `n` / `p` | 次 / 前の曲 |
| `f` | 今の曲を評価 — 押すたびに 🤔 ニュートラル → 👍 高評価 → 👎 低評価 を循環（高評価 = お気に入り、低評価だと似た曲が減ります） |
| `/` | 検索を開く |
| `l` | ライブラリを開く |
| `c` | キュー（次の曲）を開く |
| `L` | 歌詞の表示 / 非表示 |
| `d` | 今の曲をダウンロード |
| `s` | シャッフル オン / オフ |
| `r` | リピート：オフ → 全曲 → 1曲 |
| `a` | AI アシスタントを開く |
| `,` | 設定を開く |
| `?` | **すべての**キーを表示（アプリ内チートシート） |
| `Ctrl+R` | ラジオ（自動再生）オン / オフ |
| `Ctrl+H` | プレイヤーに戻る |
| `Ctrl+Q` | アプリ全体を終了 |
| `q` | 戻る / 今のパネルを閉じる |

### どのリストでも（検索・ライブラリ・キュー）

| キー | 機能 |
| --- | --- |
| `↑` / `↓` | リストを上 / 下に移動 |
| `Enter` | 選択した項目を再生（またはジャンプ） |
| `f` | お気に入り |
| `d` | ダウンロード |
| `Delete` | 削除（キューやライブラリから） |
| `q` | パネルを閉じる |

### 音にこだわる人へ

| キー | 機能 |
| --- | --- |
| `e` | イコライザのプリセットを順に切り替え |
| `N` | ラウドネス正規化のオン/オフ（静かな曲と大きい曲を揃える） |
| `>` / `<` | 再生速度を速く / 遅く |

> **日本語キーボードのヒント:** 入力を英語に切り替えなくても大丈夫です — ショートカットは韓国語の二ボル式かなも理解しますが、日本語入力（IME）中はキー入力の扱いが端末によって異なる場合があります。うまく効かないときは、いったん半角英数入力にしてみてください。

上のすべてのキーは**再割り当て可能**です — 「戻る」が `q` なのが気に入らなければ、変えてしまいましょう（設定 → Keys）。

## 自分好みにする（設定）

`,` を押して設定を開きます。上のタブは `Tab` / `Shift+Tab` で移動、`↑`/`↓` で上下に動き、`←`/`→` で値を変えます（テキスト欄は `Enter`）。終わったら `q` を押してください — 出るときに**自動保存**されます。

各タブの役割：

- **General** — Cookie ファイルの場所、ダウンロードの保存先、マウス対応オン/オフ、アルバムアートのオン/オフ、起動時の自動再生の有無、そして「リセット」ボタン2つ（1つはキーだけ、もう1つは*全部* — 散らかしすぎたとき用）。
- **Playback** — 再生速度、`←`/`→` のシークが何秒ジャンプするか、そして*ギャップレス*再生（曲間の小さな無音をなくします）。
- **EQ** — 出来合いのプリセット（Flat、Bass Boost など）付きの10バンドイコライザ、それに、ささやくほど静かな曲と壁のような爆音の曲をだいたい同じ音量にする **Normalize** トグル。
- **AI** — アシスタントのモデル選択、キーの貼り付け、自動再生**ラジオ**のオン、ラジオがどれだけ冒険的かの設定。
- **Theme** — カラーテーマを選ぶ。いくつもあります。そのうち1つはきっとあなたの魂に合います。
- **Colors** — プリセットが気に入らない？ 16進コード（`#ff8800` など）で色を1つずつ手で選べます。
- **Keys** — どのショートカットも好きなキーに再割り当て。

> **ちょっとした落とし穴：** 一部の設定には **「(next launch)」**（次回起動時）と書かれています — マウス、アルバムアート、ギャップレスなどです。これらは `ytt` を一度終了して開き直すと反映されます。正常です。アプリが無視しているわけではありません。

## ラジオ：止まらない音楽

**`Ctrl+R`** を押すと、`ytm-tui` は今あなたが聴いている曲を中心にした小さなラジオ局になります。キューが少なくなり始めると、雰囲気に合う曲をそっと並べておきます。だから音楽はただ……止まりません。怠け者の夢です。これは褒め言葉です。

どれだけ冒険的かは **設定 → AI → Radio mode** で選べます：

- **Focused** — 家の近くにとどまります。今聴いているものにより近い曲。
- **Balanced** — ほどよい中間。無難な初期値。
- **Discovery** — 道を外れて、まだ聴いたことのないものを探しに出かけます。

次の曲を*どうやって*選んでいるかは厳重に守られた企業秘密です — 正直な答えが「たくさんの試行錯誤と、ちょっとした静かな自負」だからです。とにかくちゃんと動きます。楽しんでください。

## AI アシスタント（任意）

これはおまけで、完全に任意です。無料の **Google Gemini API キー**があれば、`a` を押して音楽プレイヤーに普通の言葉で*話しかけ*られます：

- 「ローファイを流して」
- 「掃除用にアップテンポな曲を3つキューに入れて」
- 「今かかってるのは何？」
- 「これをスキップしてもっと落ち着いた曲を」

意味をくみ取って、音楽を探して、代わりにやってくれます。コマンドを覚える必要はありません。

オンにする方法：Google の AI Studio でキーを取得し（無料です）、**設定 → AI → API key** に貼り付ける。設定はそれだけです。

キーが無くても問題なし — アシスタントが眠ったままになるだけで、アプリの他のすべての機能はそれ無しでも完璧に動きます。

## ダウンロードの保存先

曲の上で `d` を押すとディスクに保存され、オフラインでも聴けます。初期の保存先はここです：

- **macOS:** `~/Music/ytm-tui`
- **Windows:** `%USERPROFILE%\Music\ytm-tui`
- **Linux:** ミュージックフォルダ内の `ytm-tui` サブフォルダ

ファイルはそのフォルダ1つに直接入ります — アーティストごとのサブフォルダを掘る必要はありません。ダウンロードした曲はライブラリの **All** と **Downloads** に自動で現れ、ストリーミングの音楽の隣に仲良く並びます。

別の場所に置きたい？ 設定の **Download dir** を変えるか、`YTM_DOWNLOAD_DIR` 環境変数を設定してください。

## Cookie でログインする（任意）

手短に：**たぶん必要ありません。** ログインしなくても、`ytm-tui` は公開されている曲を匿名でちゃんと検索・再生します。

ログインが要るのは、ログイン済みの YouTube Music アカウントが必要な曲（メンバー限定、地域制限のある曲など）にアクセスしたいときだけです。そのためには、ブラウザの「Cookie」をアプリに渡します — あなたがあなたであることを証明する小さなファイルです。

初期状態では `cookies.txt` ファイルをここで探します：

- **macOS:** `~/Music/ytm-tui/cookies.txt`
- **Windows:** `%USERPROFILE%\Music\ytm-tui\cookies.txt`

そのフォルダを作り、ブラウザから YouTube / YouTube Music の Cookie を **Netscape 形式**の `cookies.txt` として書き出して（ブラウザ拡張でできます）、そのファイルを置き、`ytt` を再起動してください。

```sh
# macOS — フォルダを作る
mkdir -p ~/Music/ytm-tui
```

```powershell
# Windows (PowerShell) — フォルダを作る
New-Item -ItemType Directory "$HOME\Music\ytm-tui" -Force
```

**設定 → Cookies file** で別のファイルを指定することもできます。

> ⚠️ **このファイルはパスワードのように扱ってください。** ログイン済みのブラウザセッションのように振る舞えるので、共有しない、リポジトリにコミットしない、どこにも投稿しないこと。

## うまくいかないとき

深呼吸。ほぼ必ずこのどれかです：

| 症状 | 対処 |
| --- | --- |
| **何も再生されない、または再生した瞬間にエラー** | mpv か yt-dlp が入っていないか、`PATH` に無いはずです。[ステップ1](#ステップ1--2つの相棒をインストール) をやり直してください。 |
| **`ytt: command not found`** | まっさらな新しいターミナルを開いてください。それでも出るなら、インストール先フォルダが `PATH` に無いということです — インストーラが追加すべき正確な行を表示したので、それをシェルの設定ファイルに貼り付けてください。 |
| **特定の曲だけ再生できない** | 一部の曲はログインが必要です。[Cookie でログインする](#cookie-でログインする任意) を見てください。 |
| **昨日まで動いたのに、今は全部再生できない** | YouTube は頻繁に変わり、yt-dlp を壊します。更新してください：`brew upgrade yt-dlp`（Mac）、`scoop update yt-dlp`（Windows）、またはパッケージマネージャ（Linux）。 |
| **アルバムアートが表示されない** | 初期状態ではオフで、一部の端末でしか動きません。設定 → General で **Album art** をオンにして**再起動**してください（「next launch」設定です）。 |
| **キーを変えたら全部めちゃくちゃになった** | 設定 → General → **Reset keybindings**。すべて水に流せます。 |

それでも詰まったら、[GitHub リポジトリ](https://github.com/Ochichan/ytm-tui) に Issue を立てて、何が起きたか書いてください。OS を教えてくれると加点です。
